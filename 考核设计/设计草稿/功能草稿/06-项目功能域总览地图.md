# 完整功能地图

## 1. 项目整体定位

本项目不是单个聊天机器人，而是一套完整的 AI Agent 平台与 AI Infra 模型调用平台。

它包含三层能力：

```text
项目 1：Agent 平台底座
项目 2：飞书超级助手机器人
项目 3：AI Infra 模型调用网关
```

整体目标：

- 支持通用 Agent 和垂直领域 Agent。
- 支持 Skill / Tool 市场和 MCP 工具接入。
- 支持多 Agent Team 协作。
- 支持 Trace、Token、配额、脱敏、告警等生产级治理能力。
- 支持飞书机器人联动，完成企业效率和告警通知场景。

## 2. 功能域总览

整个项目建议拆成 10 个功能域：

```text
0. 角色与权限体系
1. 模型与统一调用入口
2. Agent Profile 体系
3. Skill / Tool 市场
4. MCP 工具体系
5. Agent Runtime 执行引擎
6. 多 Agent Team 协作
7. 记忆系统
8. AI Infra 治理能力
9. 飞书超级助手机器人
10. 前端控制台
```

后续实现时，可以逐个功能域拆设计、建表、写接口和做页面。

## 3. 角色与权限体系

角色与权限体系贯穿整个项目。

### 3.1 角色定义

```text
Admin：平台管理员
Developer：领域智能体配置者
User：普通使用者
```

### 3.2 Admin

Admin 负责全局平台能力和安全治理。

功能：

- 用户登录和权限管理。
- 模型供应商管理。
- 全局 Skill 审核、上架、下架。
- MCP Server 管理。
- 全局安全策略配置。
- Token 配额策略配置。
- 飞书告警配置。
- 用户、应用、租户管理。
- 查看所有 Trace、Token 和告警数据。

### 3.3 Developer

Developer 负责某个应用或垂直领域智能体。

功能：

- 创建垂直领域 Agent Profile。
- 配置领域 Prompt。
- 配置默认 Skill 和可选 Skill。
- 上传领域专用 Skill。
- 配置 Team Profile。
- 查看自己负责应用的 Trace。
- 查看自己负责应用的 Token 用量。
- 配置应用级安全策略和额度。

### 3.4 User

User 是普通使用者。

功能：

- 选择通用 Agent 或垂直领域 Agent。
- 输入请求。
- 选择本次启用的 Skill。
- 上传个人私有 Skill / Tool。
- 保存个人偏好。
- 查看自己的 Trace。
- 查看自己的 Token 消耗。
- 查看自己的历史对话。

User 不能：

- 修改公共 Profile。
- 修改核心 System Prompt。
- 启用未授权高危 Skill。
- 影响其他用户。
- 绕过安全检查。

### 3.5 权限技术方案

建议采用：

```text
Spring Security + JWT + RBAC + 数据权限
```

管理端和用户端使用 JWT。

OpenAI 兼容接口使用 API Key。

## 4. 模型与统一调用入口

对应项目 1 和项目 3。

### 4.1 模型供应商管理

功能：

- 新增模型供应商。
- 编辑模型供应商。
- 删除模型供应商。
- 启用 / 禁用模型供应商。
- 查询模型供应商列表。
- 测试模型连接。

模型供应商类型：

- OpenAI
- 通义千问
- 本地模型
- OpenAI-compatible 模型

配置项：

```text
providerName
providerType
baseUrl
apiKey
modelName
enabled
defaultParams
```

### 4.2 运行时模型切换

支持：

- Agent Profile 默认模型。
- 用户请求时指定模型。
- 应用级默认模型。
- 模型不可用时降级。

### 4.3 OpenAI 兼容接口

项目 3 需要提供统一模型调用入口：

```text
POST /v1/chat/completions
```

要求：

- 请求格式兼容 OpenAI。
- 响应格式兼容 OpenAI。
- 支持 stream。
- 支持 API Key 鉴权。
- 支持多租户和应用隔离。
- 内部转发到 Agent Runtime。

### 4.4 模型调用治理

每次模型调用都需要接入：

- Trace。
- Token 统计。
- 敏感数据扫描。
- 配额控制。
- 错误告警。

## 5. Agent Profile 体系

Profile 是智能体配置，不是业务代码。

### 5.1 Profile 类型

```text
通用 Agent Profile
垂直领域 Agent Profile
Team Profile
Personal Profile
```

### 5.2 Profile 配置项

一个 Profile 可以包含：

- 名称。
- 描述。
- 模型。
- 补充 Prompt。
- 默认 Skill。
- 可选 Skill。
- MCP Tool。
- 记忆策略。
- 安全策略。
- Token 限额。
- 输出风格。
- 是否启用 Team 模式。

### 5.3 Profile 管理功能

功能：

- 新建 Profile。
- 编辑 Profile。
- 删除 Profile。
- 启用 / 禁用 Profile。
- 查询 Profile 列表。
- 复制 Profile。
- 保存个人 Profile。
- 发布垂直领域 Profile。

### 5.4 Prompt 分层

最终 Prompt 由三层组成：

```text
最终 Prompt =
平台内置核心 System Prompt
+ Profile 补充 Prompt
+ 用户本次请求 / 当前任务上下文
```

核心 System Prompt 不允许普通用户覆盖。

Developer 可以配置领域补充 Prompt。

User 可以配置个人补充 Prompt 或输出偏好。

## 6. Skill / Tool 市场

Skill / Tool 市场是项目 1 的核心能力。

### 6.1 三层 Skill

```text
全局通用 Skill
垂直领域 Skill
用户自定义 Skill
```

### 6.2 全局通用 Skill

由 Admin 维护，全平台可见。

示例：

- weather：天气。
- calculator：计算器。
- search：网络搜索。
- calendar：日历。
- translate：翻译。
- map：地图。
- file-summary：文件总结。

### 6.3 垂直领域 Skill

由 Developer 为某个领域 Agent 配置或上传。

示例：

客服助手：

- order-query。
- logistics-query。
- ticket-create。
- refund-policy。

代码审查助手：

- git-diff。
- git-log。
- ci-status。
- code-review。

飞书助手：

- feishu-schedule。
- feishu-group。
- feishu-approval。
- feishu-broadcast。

活动策划助手：

- venue-search。
- budget-plan。
- activity-template。

### 6.4 用户自定义 Skill

由普通用户上传，默认只对本人可见。

特点：

- 可在通用模式使用。
- 可作为领域模式的个人增强。
- 不影响其他用户。
- 不修改公共 Profile。
- 必须经过安全检查。

### 6.5 Skill 元数据

每个 Skill 至少包含：

```text
name
description
version
parameterSchema
returnSchema
dependencies
scope
owner
enabled
status
```

### 6.6 Skill 上传与热加载

功能：

- 上传 Jar / 配置文件。
- 保存 Skill 文件。
- 解析元数据。
- 校验参数 Schema。
- 校验版本和依赖。
- 安全检查。
- 独立 ClassLoader 加载。
- 注册到 Skill Registry。
- 启用 / 禁用。
- 卸载。

状态流转：

```text
UPLOADED
-> VALIDATING
-> INSTALLED
-> ENABLED
-> LOADED
```

异常状态：

```text
FAILED
DISABLED
UNINSTALLED
```

### 6.7 Skill 合成规则

通用模式：

```text
可用 Skill =
全局通用 Skill
+ 用户自定义 Skill
```

垂直领域模式：

```text
可用 Skill =
全局通用 Skill
+ 当前领域专用 Skill
+ 用户自定义 Skill
```

最终调用前还要经过：

```text
用户权限
应用权限
安全策略
用户本次选择
```

## 8. Agent Runtime 执行引擎

### 8.1 单 Agent 模式

```text
-> 调用模型

```

### 8.2 ReAct / Function Calling

功能：

- 最大执行步数。

- 工具结果追加到上下文。
- 终止条件判断。
- 工具异常捕获。
- 模型失败重试。

### 8.3 SSE 推送

## 9. 多 Agent Team 协作

这是项目 1 的高分项。

### 9.1 三角色设计

```text
Planner：拆任务
Executor：执行任务，调用 Skill / Tool
Reviewer：审查结果
```

### 9.2 角色约束

Planner：

- 不调用工具。
- 不直接执行任务。
- 不生成最终答案。

Executor：

- 不重新规划。
- 不修改 Planner 计划。
- 只执行分配的子任务。
- 只能调用授权 Skill / Tool。

Reviewer：

- 不重新规划。
- 不直接执行业务操作。
- 默认不调用业务 Skill / Tool。

### 9.3 Planner

输入：

- 用户请求。
- Profile。
- 可用 Skill / Tool 名称、描述、参数 Schema。
- 记忆上下文。
- Planner System Prompt。

输出：

```text
TaskPlan JSON
```

需要校验：

- JSON 合法。
- taskId 唯一。
- suggestedSkill 在可用列表中。
- dependsOn 引用存在任务。
- 无循环依赖。
- 任务数量不超限。

### 9.4 Executor

功能：

- 按依赖顺序执行任务。
- 调用授权 Skill / Tool。
- 校验参数 Schema。
- 处理工具异常。
- 输出 ExecutionResult。

调用前校验：

- 任务是否可执行。
- 依赖是否完成。
- Skill 是否授权。
- 参数是否合法。
- 是否触发敏感策略。
- 是否需要二次确认。

### 9.5 Reviewer

功能：

- 审查 Executor 执行结果。
- 判断是否完成用户目标。
- 检查遗漏、冲突和错误。
- 输出 ReviewResult。
- 必要时指定 retryTasks。

### 9.6 协作流程

```text
用户请求
-> Planner 拆解任务
-> TaskPlan 校验
-> Executor 执行任务
-> Reviewer 审查结果
-> 必要时重试任务
-> 汇总最终答案
```

### 9.7 可视化

功能：

- Mermaid 流程图。
- 任务状态面板。
- Planner / Executor / Reviewer 日志。
- 每个任务的工具调用结果。

## 10. 记忆系统

对应项目 1 的短期记忆和长期记忆要求。

### 10.1 短期记忆

功能：

- 当前会话历史。
- 滑动窗口。
- 按 token 截断。
- 注入上下文。

### 10.2 长期记忆

实现方式：

- 文件。
- Redis。
- 数据库。
- 向量库。

功能：

- 对话摘要。
- 用户偏好保存。
- 历史任务保存。
- 相关记忆检索。

### 10.3 记忆策略

```text
READ_WRITE：读写

```

## 11. AI Infra 治理能力

这是项目 3 的核心。

### 11.1 Trace

功能：

- 每次请求生成 TraceID。
- 每个阶段生成 Span。
- 记录请求进入、鉴权、模型选择、Skill 调用、MCP 调用、响应返回。
- 记录时间戳、耗时、模型、Token、状态码、错误信息。
- MDC 日志关联 TraceID。
- 本地 Trace 查询接口。
- 导出 Jaeger / Zipkin / OpenTelemetry。

### 11.2 Token 计量与配额

功能：

- 统计 prompt tokens。
- 统计 completion tokens。
- 统计 total tokens。
- 应用级日 / 月限额。
- 用户级日 / 月限额。
- 超限拦截。
- Token 用量趋势。
- TOP 消费者排行。
- 配额用尽告警。

### 11.3 敏感数据检测脱敏

检测内容：

- 手机号。
- 身份证。
- 邮箱。
- 银行卡。
- IP。
- 自定义关键词。

功能：

- 请求入站扫描。
- 响应出站扫描。
- 脱敏模式。
- 阻断模式。
- 敏感事件日志。
- 脱敏后存储，不保存原始敏感数据。

### 11.4 告警联动

告警条件：

- 模型调用失败。
- Skill 调用失败。
- MCP 调用失败。
- 错误率超阈值。
- Token 超限。
- 敏感数据阻断。
- 敏感数据疑似逃逸。
- 延迟过高。

告警功能：

- 飞书消息通知。
- 告警级别。
- TraceID 关联。
- 建议动作。
- 静默期。
- 升级策略。

### 11.5 网关拦截器链

推荐链路：

```text
TenantFilter
-> TraceFilter
-> SensitiveDataFilter
-> QuotaFilter
-> AgentRuntime
-> SensitiveResponseFilter
-> TokenRecordFilter
-> AlertFilter
```

## 16. 推荐思考顺序

后续逐个设计实现时，建议按以下顺序：

```text
1. 用户请求完整链路
2. Profile 如何存储和加载
3. Skill 如何定义、上传、加载和调用
4. Planner / Executor / Reviewer 如何交互
5. Trace / Token / 脱敏如何嵌入主链路
6. 飞书机器人如何接入告警
7. 前端页面如何展示和操作
8. Docker Compose 如何一键启动
```

这 8 个问题想清楚后，项目整体结构就会稳定。
