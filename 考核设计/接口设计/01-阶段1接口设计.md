# 阶段 1 接口设计

> 本文档定义阶段 1“基础 Agent 闭环”的接口边界。阶段 1 要跑通 `Web -> Gateway -> Core` 对话链路、SSE 流式输出、Skill/MCP 示例调用、会话消息和长期记忆持久化；不提前实现阶段 2 的完整 Trace、Token 配额、脱敏、告警和 OpenAI 兼容治理能力。

---

## 1. 文档定位

阶段 1 只设计以下接口：

```text
浏览器入口 Web API：
  注册、登录、当前用户、Application/API Key、Profile、Skill/MCP 查询、会话、浏览器 AI 对话。

Web -> Gateway 内部接口：
  浏览器 AI 请求由 Web 通过内部 HTTP 转发到 Gateway。

Gateway 阶段 1 空壳接口：
  内部 AI 执行接口、固定 SSE 测试接口、API Key 最小调用入口。

Core API Command：
  Gateway 调 Core 时使用的 AgentRunCommand / ModelInvokeCommand 字段约定。
```

阶段 1 不做：

```text
完整 OpenAI Chat Completions 兼容细节。
完整 Trace 查询接口。
完整 Token Dashboard。
完整安全策略配置接口。
完整告警规则配置接口。
完整前端 Admin 控制台接口。
Team / Planner / Executor / Reviewer 接口。
```

后续阶段 2 再补：

```text
POST /v1/chat/completions
Trace 查询
Token 当前用量 / 明细 / TOP
敏感数据策略
飞书告警接口
```

## 2. 通用约定

### 2.1 Base URL

本地开发建议：

```text
web:      http://localhost:8080
gateway:  http://localhost:8081
core:     不暴露 HTTP，由 web/gateway Maven 调用 core.api
```

说明：

- 浏览器只访问 Web。
- 外部 API Key 调用可以直接访问 Gateway。
- Web 调 Gateway 使用内部 HTTP。
- Gateway 调 Core 使用 Maven 方法调用，不是 HTTP。

### 2.2 Web 管理类响应格式

成功：

```json
{
  "success": true,
  "code": "OK",
  "message": "success",
  "data": {}
}
```

失败：

```json
{
  "success": false,
  "code": "AUTH_UNAUTHORIZED",
  "message": "未登录或登录已过期",
  "data": null
}
```

### 2.3 认证 Header

浏览器 JWT：

```text
Authorization: Bearer <jwt>
```

外部 API Key：

```text
Authorization: Bearer sk-xxx
```

Web 调 Gateway 内部接口：

```text
X-Internal-Token: <internal-token>
X-Internal-Timestamp: 1710000000000
X-Internal-Nonce: <uuid>
```

阶段 1 可以只校验固定 `X-Internal-Token`；时间戳和 nonce 先保留字段，阶段 2 再做签名和防重放。

### 2.4 分页参数

列表接口统一使用：

```text
pageNo:   从 1 开始
pageSize: 默认 20，最大 100
```

分页响应：

```json
{
  "items": [],
  "pageNo": 1,
  "pageSize": 20,
  "total": 0
}
```

### 2.5 阶段 1 错误码

| 错误码 | HTTP | 说明 |
|---|---:|---|
| `AUTH_UNAUTHORIZED` | 401 | 未登录、JWT 无效 |
| `AUTH_FORBIDDEN` | 403 | 无权限 |
| `API_KEY_INVALID` | 401 | API Key 无效或已吊销 |
| `REQUEST_INVALID` | 400 | 请求参数错误 |
| `APPLICATION_DISABLED` | 403 | 应用不可用 |
| `PROFILE_NOT_FOUND` | 404 | Profile 不存在 |
| `MODEL_CONFIG_INVALID` | 400 | 模型配置不可用 |
| `AGENT_MAX_STEPS_EXCEEDED` | 500 | Agent 超过最大循环步数 |
| `SKILL_EXECUTE_FAILED` | 500 | Skill 执行失败 |
| `MCP_TOOL_FAILED` | 500 | MCP 工具执行失败 |
| `MODEL_INVOKE_FAILED` | 502 | 模型调用失败 |
| `INTERNAL_ERROR` | 500 | 系统内部错误 |

### 6.6 Profile 使用状态

阶段 1 Profile 状态使用：

```text

DISABLED    不可用于对话
```

规则：

- 阶段 1 不做完整审核流。
- 创建者可以使用自己的 `DRAFT` Profile 发起对话。
- 非创建者只能使用 `PUBLISHED` 且自己有权限访问的 Profile。
- Core application 层必须校验 Profile 是否属于当前 `tenantId/applicationId/userId` 可访问范围，不能只靠 Web Controller 拦截。

```http

```

```text
Authorization: Bearer <jwt>

```

Request:

```json
{
  "applicationId": 20001,
```

## 10. Web -> Gateway 内部接口

Web 收到浏览器 AI 请求后，转换成内部请求调用 Gateway：

```http
POST /internal/ai/chat/stream
```

调用方：

```text
web.gatewayclient.GatewayInternalClient
```

接收方：

```text
gateway.controller.InternalAiController
```

Header:

```text
X-Internal-Token: <internal-token>
X-Internal-Timestamp: 1710000000000
X-Internal-Nonce: <uuid>
Accept: text/event-stream
```

Request:

```json
{
  "tenantId": 1,
  "applicationId": 20001,
  "userId": 10001,
  "channel": "WEB",
  "agentMode": "agent",
  "profileId": 50001,
  "conversationId": 80001,
  "message": "帮我计算 128 * 36 + 59",
  "enabledSkillIds": [60001],
  "enabledMcpToolIds": [70001],
  "clientRequestId": "uuid-from-web"
}
```

规则：

- Gateway 必须校验 `X-Internal-Token`。
- Gateway 不信任浏览器直接传来的用户身份，只信任 Web 校验后转发的内部请求。
- Gateway 将请求写入 `GatewayContext`，再显式转换成 `AgentRunCommand`。
- Core 不读取 Gateway ThreadLocal。

## 11. Gateway 固定 SSE 测试接口

Gateway 空壳完成后必须先测固定流。

```http
GET /internal/ai/stream-test
```

Header:

```text
X-Internal-Token: <internal-token>
Accept: text/event-stream
```

行为：

```text
每 500ms 输出一个 SSE 事件。
至少输出 thinking、action、observation、message、done。
```

示例输出：

```text
event: thinking
data: {"type":"thinking","traceId":"tr_test","step":1,"content":"开始处理"}

event: action
data: {"type":"action","traceId":"tr_test","step":2,"name":"calculator","content":"调用 calculator"}

event: observation
data: {"type":"observation","traceId":"tr_test","step":3,"content":"工具返回 4667"}

event: message
data: {"type":"message","traceId":"tr_test","step":4,"content":"计算结果是 4667"}

event: done
data: {"type":"done","traceId":"tr_test","step":5}
```

curl 验证：

```powershell
curl.exe -N -H "X-Internal-Token: dev-internal-token" http://localhost:8081/internal/ai/stream-test
```

验收标准：

- 终端逐条显示事件。
- 不是全部等待后一次性输出。
- Web 转发该接口时浏览器或 curl 也能逐条看到事件。

## 12. Gateway API Key 最小调用入口

阶段 1 可以先提供最小 API Key 调用入口，完整 OpenAI 兼容放阶段 2。

```http
POST /api/ai/chat/stream
```

Header:

```text
Authorization: Bearer sk-xxx
Accept: text/event-stream
```

Request:

```json
{
  "agentMode": "agent",
  "profileId": 50001,
  "modelConfigId": null,
  "conversationId": null,
  "message": "用 calculator 计算 128 * 36 + 59",
  "messages": null,
  "enabledSkillIds": [60001],
  "enabledMcpToolIds": [70001],
  "stream": true
}
```

规则：

- API Key 由 Gateway 调 `core.identity.api.ApiKeyService` 校验。
- `ApiKeyService` 在 Step 13 Gateway 实现前新增，专门负责 API Key 认证；Step 6 的 API Key 管理可暂由 `ApplicationService` 承接。
- API Key 找到 `tenantId`、`applicationId`、`userId`。
- `agentMode=agent` 调 `core.agent.api.AgentRuntimeService`。
- `agentMode=none` 调 `core.model.api.ModelInvokeService`。
- 两种模式都保留 Gateway 空壳路径。

字段规则：

```text
agentMode=agent:
  必填 profileId、message。
  conversationId 可空。
  enabledSkillIds / enabledMcpToolIds 可空。
  modelConfigId 和 messages 忽略。

agentMode=none:
  必填 modelConfigId、messages。
  不使用 profileId、conversationId、enabledSkillIds、enabledMcpToolIds。
  不走 AgentContextBuilder，不写 conversation_messages。
```

阶段 2 再新增：

```http
POST /v1/chat/completions
```

## 13. SSE 事件格式

### 13.1 通用字段

```json
{
  "type": "message",
  "traceId": "tr_xxx",
  "conversationId": 80001,
  "step": 3,
  "content": "部分回复",
  "metadata": {}
}
```

字段说明：

| 字段 | 必填 | 说明 |
|---|---|---|
| `type` | 是 | `thinking` / `action` / `observation` / `message` / `done` / `error` |
| `traceId` | 是 | 阶段 1 可由 Gateway 生成简单 TraceID |
| `conversationId` | 否 | 新会话创建后返回 |
| `step` | 是 | Agent 当前步骤 |
| `content` | 否 | 展示给用户的内容 |
| `metadata` | 否 | 工具名、耗时、错误码等附加信息 |

### 13.2 action 事件

```json
{
  "type": "action",
  "traceId": "tr_xxx",
  "conversationId": 80001,
  "step": 2,
  "content": "调用 calculator",
  "metadata": {
    "toolType": "SKILL",
    "toolName": "calculator",
    "arguments": {
      "expression": "128 * 36 + 59"
    }
  }
}
```

### 13.3 observation 事件

```json
{
  "type": "observation",
  "traceId": "tr_xxx",
  "conversationId": 80001,
  "step": 3,
  "content": "calculator 返回 4667",
  "metadata": {
    "success": true,
    "latencyMs": 12
  }
}
```

### 13.4 error 事件

```json
{
  "type": "error",
  "traceId": "tr_xxx",
  "conversationId": 80001,
  "step": 4,
  "content": "模型调用失败",
  "metadata": {
    "errorCode": "MODEL_INVOKE_FAILED",
    "retryable": true
  }
}
```

规则：

- SSE 错误也尽量通过 `error` 事件返回。
- 不把原始敏感信息写入事件内容。
- Web 转发 Gateway 错误时，也转换成 `error` 事件。

## 14. Core Command 字段约定

### 14.1 AgentRunCommand

Gateway 调 `core.agent.api.AgentRuntimeService` 时使用：

```json
{
  "tenantId": 1,
  "applicationId": 20001,
  "userId": 10001,
  "traceId": "tr_xxx",
  "channel": "WEB",
  "profileId": 50001,
  "conversationId": 80001,
  "message": "帮我计算 128 * 36 + 59",
  "enabledSkillIds": [60001],
  "enabledMcpToolIds": [70001],
  "stream": true,
  "clientRequestId": "uuid-from-web"
}
```

字段要求：

- `tenantId`、`applicationId`、`userId` 必填。
- `traceId` 阶段 1 可由 Gateway 生成。
- `channel` 为 `WEB` 或 `API_KEY`。
- `profileId` 必填。
- `conversationId` 可空。
- `enabledSkillIds` / `enabledMcpToolIds` 为空时使用 Profile 默认绑定。

### 14.2 ModelInvokeCommand

`agentMode=none` 时 Gateway 调 `core.model.api.ModelInvokeService`：

```json
{
  "tenantId": 1,
  "applicationId": 20001,
  "userId": 10001,
  "traceId": "tr_xxx",
  "channel": "API_KEY",
  "modelConfigId": 41001,
  "messages": [
    {
      "role": "user",
      "content": "你好"
    }
  ],
  "temperature": 0.7,
  "stream": true
}
```

规则：

- 纯模型透传不走 AgentContextBuilder。
- 仍经过 Gateway 空壳路径。
- 阶段 2 再完整接入 Token、Trace、脱敏治理。

## 16. 与后续阶段的关系

阶段 1 接口完成后，阶段 2 在 Gateway 上增强：

```text

Trace Root / Span 创建与查询

飞书告警事件
```

阶段 1 不应为了阶段 2 预先暴露大量空接口。只需要保证：

```text

Gateway 能生成 traceId 并传入 Core。
Core Command 包含 tenant/application/user/channel/profile/conversation。
SSE 链路已经验证不缓冲。
```
