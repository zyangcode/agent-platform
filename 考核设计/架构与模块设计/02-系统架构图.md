## 2. 请求主链路时序图

```mermaid
sequenceDiagram
    actor U as 用户/代码
    participant WEB as Web(:8080)
    participant GW as Gateway(:8081)
    participant TF as TenantFilter
    participant TR as TraceFilter
    participant SD as SensitiveDataFilter
    participant QF as QuotaFilter
    participant RT as AgentRuntime(core)
    participant ML as ModelClient
    participant SK as SkillEngine
    participant SR as SensitiveResponseFilter
    participant TK as TokenRecordFilter
    participant AL as AlertFilter
    participant FS as feishu-bot

    alt 浏览器 AI 对话/测试
        U->>WEB: POST /api/chat<br/>JWT
        WEB->>GW: 内部转发 AI 执行请求<br/>user_id/app_id/tenant_id<br/>内部服务 Token / 签名 Header
    else 外部 API Key 调用
        U->>GW: POST /v1/chat/completions<br/>Authorization: Bearer sk-xxx
    end

    activate GW
    GW->>TF: 解析 API Key 或 Web 转发身份<br/>→ app_id/user_id/tenant_id
    TF->>TR: GatewayContext 写入
    TR->>TR: 生成 traceId + root Span<br/>MDC.put("traceId",...)
    TR->>SD: 入站扫描
    SD->>SD: 正则匹配敏感数据<br/>→ PASS/MASK/BLOCK
    SD->>QF: 进入配额检查
    QF->>QF: Redis/DB 原子预扣
    QF->>RT: 配额通过，进入引擎

    activate RT
    RT->>RT: 加载 Profile
    RT->>RT: 加载记忆（短期窗口+长期检索）
    RT->>RT: Skill 合成（全局+领域+个人→过滤黑白名单）
    RT->>RT: 构建上下文（System Prompt+历史+工具列表）

    loop ReAct while 循环
        RT->>ML: Think: 调 LLM
        ML-->>RT: 返回：需调工具 / 直接回复
        alt 需调工具
            RT->>SK: Act: 调 Skill
            SK-->>RT: 返回工具结果
            RT->>RT: Observe: 结果回填 messages
        else 直接回复
            RT->>RT: 结束循环
        end
    end

    RT-->>GW: AgentResult
    deactivate RT

    GW->>SR: 出站扫描
    SR->>SR: 扫描响应内容 → 脱敏
    SR->>TK: Token 结算
    TK->>TK: 按真实 usage 修正扣减<br/>写入 token_usage_log
    TK->>AL: 告警判断
    AL->>AL: 错误？超限？阻断？
    alt 需告警
        AL->>AL: 写告警事件 / 队列
        AL-->>FS: 异步 HTTP 调 /internal/alerts/feishu
        FS->>FS: 发送飞书卡片消息
    end
    AL->>GW: 响应就绪

    deactivate GW
    GW-->>U: X-Trace-Id: trace-xxx<br/>SSE 流 / JSON 响应
```

## 3. 模块依赖方向图

```mermaid
graph LR
    subgraph 入口
        Web[web<br/>:8080<br/>浏览器/管理入口]
        Gateway[gateway<br/>:8081<br/>统一 AI 调用入口]
    end

    subgraph 引擎
        Core[core]
    end

    subgraph 基础
        Common[common<br/>异常/错误码/工具]
    end

    subgraph 独立
        Feishu[feishu-bot<br/>:8082]
    end

    subgraph 前端
        FE[frontend<br/>React]
    end

    FE -->|HTTP REST| Web
    Web -->|管理配置查询| Core
    Web -->|AI 对话/测试转发| Gateway
    Gateway -->|AI 执行/模型透传| Core
    Gateway -.->|异步 HTTP 告警接口| Feishu
    Core --> Common
```

**禁止依赖**：core → web、core → gateway、common → 任何业务模块

说明：feishu-bot 是独立服务，不依赖 agent-platform 的 common/core/web/gateway 代码；这里只允许通过 HTTP 告警接口通信。

## 4. 拦截器链顺序

```mermaid
graph LR
    A[请求进入] --> B[TenantFilter<br/>租户识别]
    B --> C[TraceFilter<br/>TraceID+rootSpan]
    C --> D[SensitiveDataFilter<br/>入站敏感扫描]
    D --> E[QuotaFilter<br/>Token配额预扣]
    E --> F[RuntimeInvokeFilter<br/>调用 core AgentRuntime]
    F --> G[SensitiveResponseFilter<br/>出站敏感扫描]
    G --> H[TokenRecordFilter<br/>Token结算修正]
    H --> I[AlertFilter<br/>告警条件判断]
    I --> J[响应返回]

    style F fill:#4a90d9,color:#fff
    style B fill:#888
    style C fill:#888
    style D fill:#888
    style E fill:#888
    style G fill:#888
    style H fill:#888
    style I fill:#888
```

全部 Filter 共享 GatewayContext（ThreadLocal），请求结束在 finally 中清理。

## 5. Skill 状态流转图

```mermaid
stateDiagram-v2
    [*] --> UPLOADED : 上传 Jar/配置
    UPLOADED --> VALIDATING : 自动校验
    VALIDATING --> INSTALLED : 校验通过
    VALIDATING --> FAILED : 校验失败
    INSTALLED --> ENABLED : Admin 启用
    INSTALLED --> DISABLED : Admin 禁用
    ENABLED --> LOADED : 首次调用/热加载
    LOADED --> DISABLED : 热卸载
    DISABLED --> ENABLED : 重新启用
    DISABLED --> UNINSTALLED : 卸载
    FAILED --> UPLOADED : 重新上传
    UNINSTALLED --> [*]
```
