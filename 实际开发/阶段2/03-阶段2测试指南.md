## 5. Web Chat Trace 验收

调用：

```powershell
curl.exe -N `
  -H "Authorization: Bearer <jwt>" `
  -H "Content-Type: application/json" `
  -d '{"applicationId":1,"profileId":1,"message":"hello","agentMode":"agent","stream":true}' `
  http://localhost:8080/api/chat/stream
```

记录 SSE 中的：

```text
traceId = tr_xxx
conversationId = n
```

成功标准：

```text
SSE 先返回 request accepted。
最终返回 message 和 done。
done 前后没有长时间卡死。
traceId 格式为 tr_ + 32 位 UUID hex。
```

## 6. DBeaver Trace 验收

替换 traceId：

```sql
select *
from trace_roots
where trace_id = 'tr_xxx';
```

成功标准：

```text
有 1 条记录。
status = SUCCESS。
entrypoint = INTERNAL_WEB。
tenant_id/application_id/user_id/profile_id 有值。
conversation_id 有值。
latency_ms >= 0。
```

查询 spans：

```sql
select span_name, span_type, component, status, latency_ms, attributes
from trace_spans
where trace_id = 'tr_xxx'
order by started_at asc;
```

成功标准：

```text
至少包含 agent_runtime.run。
至少包含 context.build。
至少包含 model.invoke。
所有成功 span 状态为 SUCCESS。
latency_ms >= 0。
```

查询 token：

```sql
select model_name,
       provider_type,
       prompt_tokens,
       completion_tokens,
       total_tokens,
       estimated,
       created_at
from token_usage_logs
where trace_id = 'tr_xxx'
order by created_at asc;
```

成功标准：

```text
至少 1 条记录。
total_tokens > 0。
mock-chat 的 estimated = true。
真实模型如果 provider 返回 usage，estimated = false；如果没有返回 usage，estimated = true。
```

## 10. Trace 写入失败降级验收

该项可以通过单元测试优先覆盖，不一定手工破坏数据库。

建议测试：

```text
TraceService.startRoot 写库异常时，Gateway 仍返回 SSE。
TraceService.finishRoot 写库异常时，SSE done 不受影响。
TokenUsageService.record 写库异常时，AgentRuntime 仍返回 AgentRunResult。
```

成功标准：

```text
Chat 主链路不因为普通 Trace/Usage 明细写入失败而失败。
日志中能看到 warn。
```

注意：

```text
这条规则不适用于后续真实 Token 配额预扣。
配额预扣失败必须阻断请求并返回 429。
```

## 11. Token 配额预扣验收

阶段 2.2 的自动化测试优先覆盖：

```text
DefaultQuotaServiceTest:
  无 quota_configs 配置时 reserve 默认允许并插入 RESERVED reservation。
  single_request_limit_tokens 超限时抛 QUOTA_EXCEEDED。
  commit 只更新 RESERVED reservation 为 COMMITTED。
  release 只更新 RESERVED reservation 为 RELEASED。
  reservation 已经终态时重复 release 不抛异常。

InternalAiControllerTest:
  reserve 超限时返回 HTTP 429。
  reserve 超限时不调用 AgentRuntime。
  AgentRuntime 成功后 commit。
  AgentRuntime 失败后 release。
```

手工验收建议：

```text
插入一条 quota_configs，subject_type=APPLICATION，subject_id=当前 application_id，single_request_limit_tokens 小于默认预估值 1000。
调用 /internal/ai/chat/stream。
预期返回 HTTP 429 + QUOTA_EXCEEDED。
删除或禁用该 quota_configs 后再次调用。
预期 SSE 正常返回 message/done，quota_reservations 产生 COMMITTED 记录。
```

## 12. 阶段 2 验收结论模板

```text
本次请求 traceId: tr_xxx
入口: INTERNAL_WEB / API_KEY

trace_spans: n 条，包含 agent_runtime.run、context.build、model.invoke
token_usage_logs: 1 条，total_tokens > 0
quota_reservations: 1 条，状态 COMMITTED；失败请求为 RELEASED；超限请求不进入 AgentRuntime
mock-chat: estimated = true
DeepSeek: estimated = false，或 provider 未返回 usage 时 estimated = true
Trace 查询接口: GET /api/traces/{traceId} 通过
全量测试: mvn.cmd -s .mvn/settings.xml test BUILD SUCCESS
```
