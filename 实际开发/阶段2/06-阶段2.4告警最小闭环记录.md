# 阶段 2.4 告警最小闭环记录

## 1. 本轮目标

阶段 2.4 先做平台侧告警事件闭环：Gateway 在关键治理事件发生时写入 `alert_events`，后续由通知适配器或异步 worker 调用 feishu-bot。

本轮不做真实飞书 HTTP 发送，避免外部服务影响 AI 主链路。

## 3. 触发范围

第一版触发三类告警：

```text
SECURITY_BLOCKED  敏感数据阻断，level=WARN
TOKEN_EXCEEDED    Token 配额超限，level=WARN
MODEL_ERROR       AgentRuntime / Model 调用失败，level=ERROR
```

当前执行点：

```text
SensitiveDataFilter.scanRequest 异常或阻断后，写 SECURITY_BLOCKED。
QuotaFilter.reserve 抛 QUOTA_EXCEEDED 后，写 TOKEN_EXCEEDED。
SSE 执行中 AgentRuntime / ModelInvoke 抛异常后，写 MODEL_ERROR。
```

## 4. 失败策略

告警事件属于边缘治理动作，不能影响用户请求：

```text
AlertEventService 写库失败只记录 warn，不抛出。
AlertFilter 调用 AlertEventService 失败只记录 warn，不改变原 HTTP/SSE 响应。
429 仍然返回 QUOTA_EXCEEDED。
403 仍然返回 SECURITY_BLOCKED。
AgentRuntime 失败仍然返回 SSE error。
```
