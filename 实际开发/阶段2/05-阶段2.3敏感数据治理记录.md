# 阶段 2.3 敏感数据治理记录

## 1. 本轮目标

阶段 2.3 先做 Gateway 请求入口阻断，保证明显敏感数据不会进入 AgentRuntime、模型调用、长期记忆和 Token 配额结算链路。

本轮只扫描：

```text
GatewayChatRequest.message
location = REQUEST_MESSAGE
```

本轮不扫描：

```text
模型响应
工具入参
Skill/MCP 返回结果
记忆写入内容
Trace attributes
```

## 3. 拦截顺序

当前阶段的实际执行顺序：

```text
InternalAiController
  -> generate traceId
  -> SensitiveDataFilter.scanRequest(...)
  -> QuotaFilter.reserve(...)
  -> AgentRuntimeService.run(...) / ModelInvokeService.invoke(...)
```

关键点：

```text
敏感数据命中时返回 HTTP 403 + SECURITY_BLOCKED。
敏感数据命中时不执行 QuotaFilter.reserve。
敏感数据命中时不调用 AgentRuntimeService.run。
扫描器异常采用 fail-close，也返回 HTTP 403。
security_events 记录失败只写 warn，不影响阻断结果。
```

## 4. 规则与存储

第一版规则使用正则列表实现，不用 if-else 分散扫描逻辑：

```text
PHONE
EMAIL
ID_CARD
API_KEY_PATTERN
```

security_events 不保存敏感原文，只保存：

```text
event_type
location
source_text_hash
masked_sample
action = BLOCK
```

`source_text_hash` 为敏感原文的 SHA-256，用于去重和排查；`masked_sample` 只保存脱敏样例。
