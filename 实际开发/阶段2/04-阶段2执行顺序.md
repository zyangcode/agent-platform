# 阶段 2 执行顺序

> 本文档把阶段 2.1 Trace + Token Usage 闭环拆成可连续执行的小步骤。每个 Step 都应先写测试，再实现，再跑对应测试和全量 `mvn.cmd -s .mvn/settings.xml test`。阶段 2.2 配额预扣、2.3 脱敏、2.4 告警不放进本轮。

---

## 1. 本轮目标

实现阶段 2.1：

```text
V006 trace/token 表。
core.trace/core.quota 包。
Gateway 创建并结束 trace_root。
AgentRuntime 记录 agent/context/model span。
AgentRuntime 记录 token_usage_logs。
Web 查询 trace 详情和列表。
失败请求写 FAILED。
Trace/Usage 普通写入失败不影响 Chat。
```

不做：

```text
Token 配额预扣和 429。
敏感数据脱敏。
飞书告警。
完整 OpenAI /v1/chat/completions。
前端页面。
Graph/WebFlux/Reactor。
Agent Runtime Pipeline 重构。
```

阶段 2 实现原则：

```text
阶段 2 不先做 Agent Runtime Pipeline 重构。
DefaultAgentRuntimeService 只允许新增小范围 Trace/Token 包装方法，例如 startSpan/finishSpan/recordTokenUsage。
如果阶段 2 完成后 run()/runModelLoop 明显膨胀，再单独做 Pipeline 重构。
LLM Provider Registry 作为 core.model 内部增强优先落地，对外仍只暴露 ModelInvokeService。
```


## 10. Step 9：AgentRuntime 记录 span 和 token usage

修改：

```text
agent-platform-core/src/main/java/com/ls/agent/core/agent/application/DefaultAgentRuntimeService.java
```

注入：

```text
core.trace.api.TraceService
core.quota.api.TokenUsageService
```

实现：

```text
run 开始时 startSpan(agent_runtime.run)。
contextBuilder.build 外包 startSpan(context.build)/finishSpan。
每次 modelInvokeService.invoke 外包 startSpan(model.invoke)/finishSpan。
拿到 ModelInvokeResult 后 record token_usage_logs。
run 正常结束 finishSpan(agent_runtime.run, SUCCESS)。
异常时 finishSpan(agent_runtime.run, FAILED) 后继续抛业务异常给 Gateway。
```

model.invoke span attributes：

```text
modelConfigId
step
```

token usage 字段：

```text
traceId 从 AgentRunCommand 取。
tenantId/applicationId/userId/profileId 从 AgentRunCommand 取。
modelConfigId/modelName/usage 从 ModelInvokeResult 取。
providerId/providerType 第一版如果 ModelInvokeResult 没有，可通过 TraceService/TokenUsageService 内部查 model_config/model_provider，或先扩展 ModelInvokeResult。
```

推荐第一版：

```text
TokenUsageService.record 入参要求 providerId/providerType。
DefaultAgentRuntimeService 不通过 ModelConfigService 反查 provider。
ModelInvokeResult 必须已经包含 providerId/providerType/modelName。
```

测试：

```text
agent-platform-core/src/test/java/com/ls/agent/core/agent/DefaultAgentRuntimeServiceTest.java
agent-platform-core/src/test/java/com/ls/agent/core/model/DefaultModelInvokeServiceTest.java
```

覆盖：

```text
run records agent/context/model spans。
run records token usage。
model loop multiple calls records multiple model.invoke spans。
model failure marks spans FAILED and rethrows。
trace service failure does not break successful agent run。
```
