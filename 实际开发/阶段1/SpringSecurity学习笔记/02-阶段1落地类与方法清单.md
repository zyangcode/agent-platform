# 阶段 1 Spring Security + JWT 落地类与方法清单

本文把阶段 1 要写的类、方法、职责先列清楚，后面实现时可以按这个顺序推进。

## 1. 实现顺序

```text
1. users 表、UserEntity、UserMapper
2. PasswordEncoder
3. core.identity.IdentityAuthService
4. web.auth.AuthController / AuthAppService
5. JwtProperties
6. JwtTokenService
7. CurrentUser / JwtClaims
8. JwtAuthenticationFilter
9. RestAuthenticationEntryPoint / RestAccessDeniedHandler
10. SecurityConfig
11. 接口测试
```

## 2. core.identity

### IdentityAuthService

位置：

```text
agent-platform-core
com.ls.agent.core.identity.api.IdentityAuthService
```

方法建议：

```java
RegisterResult register(RegisterCommand command);

LoginUserResult verifyPassword(LoginCommand command);

UserDetailResult getUserDetail(Long userId);
```

职责：

```text
注册用户
校验用户名密码
查询用户详情
判断用户状态
```

### IdentityAuthServiceImpl

位置：

```text
com.ls.agent.core.identity.application.IdentityAuthServiceImpl
```

职责：

```text
调用 UserMapper
使用 PasswordEncoder 加密/校验密码
返回 core 层结果对象
```

注意：

```text
这里不生成 JWT。
JWT 属于 web.security。
```

### UserMapper

方法建议：

```java
UserEntity selectByUsername(String username);

UserEntity selectById(Long id);

int insert(UserEntity entity);
```

阶段 1 最小够用。

## 3. web.auth

### AuthController

接口：

```text
POST /api/auth/register
POST /api/auth/login
GET  /api/auth/me
```

方法建议：

```java
RegisterResponse register(RegisterRequest request);

LoginResponse login(LoginRequest request);

CurrentUserResponse me(@AuthenticationPrincipal CurrentUser currentUser);
```

### AuthAppService

职责：

```text
把 Web Request 转成 core Command
调用 IdentityAuthService
登录成功后调用 JwtTokenService 生成 token
组装 LoginResponse
```

方法建议：

```java
RegisterResponse register(RegisterRequest request);

LoginResponse login(LoginRequest request);
```

## 4. web.security

### JwtProperties

配置项建议：

```text
security.jwt.secret
security.jwt.issuer
security.jwt.expire-seconds
```

字段：

```java
private String secret;
private String issuer;
private long expireSeconds;
```

### JwtTokenService

方法建议：

```java
String generateToken(CurrentUser currentUser);

JwtClaims parseToken(String token);

boolean validateToken(String token);
```

职责：

```text
生成 JWT
校验签名
校验过期时间
解析 userId / tenantId / username / roles
```

### JwtClaims

字段建议：

```java
Long userId;
Long tenantId;
String username;
List<String> roles;
Instant issuedAt;
Instant expiresAt;
```

方法建议：

```java
CurrentUser toCurrentUser();

Collection<GrantedAuthority> toAuthorities();
```

### CurrentUser

字段建议：

```java
Long userId;
Long tenantId;
String username;
List<String> roles;
```

用途：

```text
作为 Authentication principal。
Controller 用 @AuthenticationPrincipal 获取。
```

### JwtAuthenticationFilter

继承：

```java
OncePerRequestFilter
```

核心方法：

```java
protected void doFilterInternal(HttpServletRequest request,
                                HttpServletResponse response,
                                FilterChain filterChain)
```

逻辑：

```text
读取 Authorization Header
判断是否 Bearer token
调用 JwtTokenService 校验
解析 CurrentUser
构造 UsernamePasswordAuthenticationToken
写入 SecurityContextHolder
继续 filterChain.doFilter
```

注意：

```text
Filter 不查数据库。
Filter 不写业务权限。
Filter 只恢复登录态。
```

### SecurityConfig

职责：

```text
关闭 CSRF
关闭 Session
配置放行接口
配置 ADMIN 接口
注册 JWT Filter
配置 401/403 处理器
提供 PasswordEncoder
```

阶段 1 放行：

```text
/api/auth/register
/api/auth/login
/api/chat/stream-test
```

需要 ADMIN：

```text
/api/admin/**
```

其余：

```text
authenticated
```

### RestAuthenticationEntryPoint

触发条件：

```text
未登录访问受保护接口
```

返回：

```json
{
  "code": "UNAUTHORIZED",
  "message": "请先登录"
}
```

HTTP 状态：

```text
401
```

### RestAccessDeniedHandler

触发条件：

```text
已登录但权限不足
```

返回：

```json
{
  "code": "FORBIDDEN",
  "message": "无权限访问"
}
```

HTTP 状态：

```text
403
```

## 5. 测试用例顺序

### 5.1 注册

```http
POST /api/auth/register
Content-Type: application/json

{
  "username": "zhangsan",
  "password": "123456"
}
```

检查：

```text
users 表有记录
password 字段不是明文
```

### 5.2 登录

```http
POST /api/auth/login
Content-Type: application/json

{
  "username": "zhangsan",
  "password": "123456"
}
```

检查：

```text
返回 accessToken
tokenType = Bearer
```

### 5.3 获取当前用户

```http
GET /api/auth/me
Authorization: Bearer token
```

检查：

```text
返回 userId / username / role
```

### 5.4 不带 token

```http
GET /api/auth/me
```

检查：

```text
401
```

### 5.5 权限不足

```http
GET /api/admin/model-configs
Authorization: Bearer USER_TOKEN
```

检查：

```text
403
```

## 6. 关键判断

如果后面实现时出现这些情况，要停下来调整：

```text
Filter 直接查 UserMapper
JWT 逻辑放进 core
Controller 直接查 UserMapper
core 依赖 web.security.CurrentUser
父 POM 把 spring-boot-starter-security 放进所有模块
SecurityConfig 里堆业务权限规则
```

这些都会让边界变乱。
