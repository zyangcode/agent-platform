## 3. 执行步骤

### Step 1：创建 Maven 多模块工程

创建：

```text
agent-platform-common
agent-platform-core
agent-platform-web
agent-platform-gateway
```

父 POM 约束：

```text
父 POM 使用 dependencyManagement / pluginManagement 统一管理依赖版本和插件版本。
业务依赖放到各子模块 dependencies，避免父 POM dependencies 污染所有模块。
Spring Boot BOM / Spring AI BOM 在父 POM dependencyManagement 中导入。
Spring Boot 使用 3.4.x 版本线，保证与 Spring AI 1.0.x 兼容。
```

验收：

```text
mvn test 能跑通。
frontend 不加入 Maven modules。
feishu-bot 不依赖 agent-platform 代码。
```

### Step 2：定义 common 基础能力

实现：

```text
ApiResponse
PageResult
ErrorCode
BizException

```

禁止：

```text
common 放业务 DTO
common 放 Entity
common 放 Mapper
```

验收：

```text
common 不依赖 core/web/gateway。
```

验收：

### Step 5：加入 ArchUnit

最低规则：

```text
web/gateway 禁止访问 core.*.mapper

Controller 禁止直接访问 Mapper
common 禁止包含业务 DTO/Entity/Mapper/Service
```

验收：

```powershell
mvn test
```

必须包含 ArchUnit 测试。

### Step 6：实现 identity

实现接口：

```text
POST /api/auth/register
POST /api/auth/login
GET /api/auth/me
POST /api/applications
GET /api/applications
GET /api/applications/{applicationId}/api-keys
POST /api/applications/{applicationId}/api-keys/{apiKeyId}/revoke
```

Core 能力：

```text
AuthService
ApplicationService
```

说明：

```text
Step 6 当前已落地 AuthService / ApplicationService。
API Key 创建、查询、吊销暂由 ApplicationService 承接。
Gateway API Key 校验进入 Step 13 前再新增 ApiKeyService。
PermissionService 暂不单独拆，资源归属校验先放各业务 Application Service。
```

Web 安全能力：

```text
SecurityConfig
JwtAuthenticationFilter
JwtTokenService
JwtProperties
CurrentUser
RestAuthenticationEntryPoint
RestAccessDeniedHandler
PasswordEncoder / BCryptPasswordEncoder
```

验收：

```text
Apifox 能注册、登录、创建 Application。
登录后返回 JWT。
GET /api/auth/me 能通过 Authorization: Bearer <jwt> 返回当前用户。
未登录访问受保护接口返回 401 JSON。
USER 访问 /api/admin/** 返回 403 JSON。
创建 Application 时返回一次 API Key 明文。
DBeaver 中 api_keys 只有 hash 和 prefix。
```


### Step 13：实现 gateway 空壳

实现：

```text
GET /internal/ai/stream-test
POST /internal/ai/chat/stream
POST /api/ai/chat/stream
```

要求：

```text
校验 X-Internal-Token。
API Key 调 core.identity.api.ApiKeyService 校验；如果 Step 6 尚未拆出该接口，本步骤先补齐。
生成简单 traceId。
GatewayContext finally 清理。
显式转换 AgentRunCommand / ModelInvokeCommand。
agentMode=agent 调 core.agent.api.AgentRuntimeService。
agentMode=none 调 core.model.api.ModelInvokeService。
不访问 core.*.mapper。
```

验收：

```powershell
curl.exe -N -H "X-Internal-Token: dev-internal-token" http://localhost:8081/internal/ai/stream-test
```

终端逐条输出 SSE 事件。

### Step 14：实现 web 最小接口与 SSE 透传

实现：

```text
补齐并联调 Web Auth / Application / Model / Profile / Skill / MCP / Conversation Controller
GET /api/chat/stream-test
POST /api/chat/stream
```

要求：

```text
前面已实现的 Web Controller 在本步骤统一联调。
Web AI 请求不能直接调 core.agent。
Web 通过 HTTP 调 Gateway。
Web 逐事件透传 Gateway SSE。
Gateway 出错时 Web 转成 SSE error 事件。
```

验收：

```powershell
curl.exe -N -H "Authorization: Bearer <jwt>" http://localhost:8080/api/chat/stream-test
```

终端逐条输出事件。

## 4. 阻塞判断

遇到阻塞时按下面处理：

```text
模型不可用：
  先用 mock model，不能阻塞主链路。

天气 / 搜索外部 API 不可用：
  使用 mock Skill 返回。

MCP 工具接入复杂：
  先做只读本地示例工具。

SSE 被缓冲：
  先停下修 Web -> Gateway 透传，不继续写 Agent。

ArchUnit 拦截越界：
  优先调整包依赖，不绕过测试。
```
