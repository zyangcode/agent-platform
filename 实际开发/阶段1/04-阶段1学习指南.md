# 阶段 1 学习指南

> 从一根 HTTP 请求追到数据库，理解 Spring Boot 多模块项目里每一层在做什么、每一行代码为什么放这里。

---

## 一、阶段 1 做了什么

一句话：**搭建一个能注册、登录、配置模型和 Agent Profile、发起 AI 对话、SSE 实时返回的基础 Agent 平台。**

```text
用户注册 → 登录拿 JWT → 创建 Application（拿 API Key）
  → 配置模型 → 创建 Agent Profile → 绑定 Skill/MCP
  → 发起对话 → Web 转发 Gateway → Agent Runtime（ReAct 循环）
  → SSE 实时返回 → 保存会话 → 写入长期记忆
```

83 个测试覆盖全部 14 个 Step，零失败。

---

## 二、核心架构：为什么这么拆

### 2.0 先记住这些术语

```text
Tenant：租户隔离单位，所有核心数据都带 tenantId。
User：平台用户，登录后通过 JWT 访问 Web 接口。
Application：用户创建的应用，API Key 绑定在 Application 上。
API Key：外部程序调用 Gateway 的凭证，只在创建时展示明文。
Profile：Agent 配置，包含模型、Prompt、可用 Skill/MCP、记忆策略等。
Skill：平台内置或上传的工具能力，例如 calculator/weather/search。
MCP Tool：通过 MCP 协议接入的外部工具能力。
Gateway：AI 请求统一入口，负责鉴权、治理链和协议适配。
Agent Runtime：真正执行 Agent 对话、ReAct 循环和工具调用的核心服务。
Context：运行时拼出来的模型输入，包含 system prompt、历史消息、记忆和工具列表。
Memory：长期记忆，阶段 1 用数据库做简单持久化和召回。
```

### 2.1 四个 Maven 模块

```text
agent-platform/
├── common/      公共基础（谁都能用）
├── core/        核心业务（13 个包，有表、有逻辑）
├── web/         浏览器入口（JWT + 管理接口 + 转发 Gateway）
└── gateway/     AI 调用入口（API Key + 治理链 + SSE）
```

**关键约束**：
- `web → gateway` 只能走 HTTP，不能走 Maven 依赖
- `gateway → core` 走 Maven 方法调用
- `web / gateway` 没有业务表，不能访问 `core.*.mapper`
- `common` 不能放 Entity、DTO、Service

### 2.2 为什么这么拆

```text
如果把 web、gateway、core 全放一个 Maven 模块：
  Controller 可以直接调 Mapper → 编译器不报错 → 架构边界只能靠人审

拆成独立 Maven 模块：
  web 想调 gateway 的代码？ → 编译失败，web 没依赖 gateway
  web/gateway 虽然依赖 core，但只能使用 core 的 api/dto/command
  web/gateway 如果访问 core.*.mapper / entity / internal → 编译可能过，但 ArchUnit 测试会失败
```

**一句话：Maven 模块边界用编译器守卫；web/gateway 到 core 的细边界、core 内部包边界，用 ArchUnit 测试守卫。**

## 三、最快理解路径：先追三条链路

如果你想快速理解整个阶段 1，不要先从所有表和所有测试开始。先追这三条链路：

### 链路 1：注册登录

```text
AuthController
  -> AuthService
  -> DefaultAuthService
  -> UserMapper / ApplicationMapper / ApiKeyMapper
  -> JwtTokenService
  -> JwtAuthenticationFilter
  -> CurrentUserArgumentResolver
```

看完这条链路，你会理解用户怎么进系统、JWT 怎么生成和解析、Controller 怎么拿到当前用户。

### 链路 2：创建 Profile

```text
ProfileController
  -> ProfileService
  -> DefaultProfileService
  -> ApplicationService
  -> SkillQueryService
  -> McpToolQueryService
```

看完这条链路，你会理解 core 内部跨包调用纪律：Profile 要校验 Application、Skill、MCP，但只能调对应包的 api，不能直接碰 mapper。

### 链路 3：AI 对话

```text
ChatController
  -> HttpGatewayClient
  -> InternalAiController
  -> AgentRuntimeService
  -> AgentContextBuilder
  -> ModelInvokeService
  -> SkillExecutor / McpToolExecutor
  -> MemoryWriteService
```

看完这条链路，你会理解为什么 AI 请求必须经过 Gateway、SSE 怎么透传、Agent Runtime 怎么组装上下文和调用工具。

## 四、学习路线（按执行顺序）

### 第 1 天：看骨架怎么搭

**读这些文件**：
1. `pom.xml`（根目录）—— 看 `<dependencyManagement>` 怎么管版本、`<modules>` 怎么聚合
2. `agent-platform-common/pom.xml` —— 只有测试依赖，没有业务依赖
3. `agent-platform-core/pom.xml` —— MyBatis-Plus + Flyway + PostgreSQL
4. `agent-platform-web/pom.xml` —— 依赖 core + common，**不依赖 gateway**
5. `agent-platform-gateway/pom.xml` —— 依赖 core + common

**核心知识点**：
```text
<dependencyManagement>  只管版本，不引入依赖
<dependencies>          才真引入依赖
<modules>               聚合构建（父 POM 知道有哪些子模块）
<parent>                继承（子模块知道父 POM 是谁）
```

### 第 2 天：看数据怎么存

**读这些文件**：
1. `agent-platform-core/src/main/resources/db/migration/V001~V005` —— 18 张表的 CREATE 语句
2. `agent-platform-core/src/main/java/com/ls/agent/core/support/persistence/BaseEntity.java`
3. `agent-platform-core/src/main/java/com/ls/agent/core/identity/entity/UserEntity.java`
4. `agent-platform-core/src/main/java/com/ls/agent/core/identity/mapper/UserMapper.java`

**核心知识点**：
```text
Flyway:    应用启动自动执行 SQL，版本号双下划线 V001__xxx.sql
MyBatis-Plus: 继承 BaseMapper<XxxEntity> → CRUD 全自动生成
Entity:    继承 CreatedEntity（有 createdAt）
           继承 BaseEntity（有 createdAt + updatedAt）
           继承 VersionedEntity（有 createdAt + updatedAt + version）
           继承 SoftDeleteEntity（有 createdAt + deletedAt）
jsonb 字段: @TableField(typeHandler = JsonNodeTypeHandler.class)
           + @TableName(autoResultMap = true)
ArchUnit:  每个 Entity/Mapper 生成后立刻有架构测试守卫边界
```

### 第 3 天：看用户怎么登录

**读这些文件（按请求顺序）**：

```text
请求进来 POST /api/auth/register
  │
  ▼
1. web/controller/AuthController.java          ← 接收 HTTP 请求，调 AuthService
2. core/identity/api/AuthService.java           ← 接口（定义能干什么）
3. core/identity/application/DefaultAuthService.java  ← 实现（真正写用户表）
4. core/identity/mapper/UserMapper.java         ← 空的！extends BaseMapper
  │
  ▼
请求进来 POST /api/auth/login
  │
  ▼
5. web/controller/AuthController.login()        ← 调 AuthService.login()
6. web/security/JwtTokenService.java            ← 生成 JWT
7. web/security/JwtAuthenticationFilter.java    ← 每次请求解析 JWT
8. web/security/CurrentUserArgumentResolver.java ← 把 CurrentUser 注入 Controller 参数
```

**核心知识点**：
```text
密码:   BCryptPasswordEncoder.encode() 存 → matches() 校验
JWT:    jjwt 0.12.x 生成 → 存 userId/tenantId/roles → HMAC 签名
Security: 无状态（STATELESS），JwtAuthenticationFilter 从 Header 取 token 解析
CurrentUser: Record 类型，ThreadLocal 传递：
  Filter 写 SecurityContextHolder → ArgumentResolver 读 → Controller 直接用
API 返回: ApiResponse<T>（Record）+ PageResult<T>（Record）
错误处理: BizException(ErrorCode) → GlobalExceptionHandler → JSON 错误
```

### 第 4 天：看模型怎么配置

**读这些文件**：
1. `core/model/api/ModelConfigService.java` —— 管理 provider + config 的接口
2. `core/model/application/DefaultModelConfigService.java` —— 实现
3. `core/model/api/ModelInvokeService.java` —— 调模型的接口
4. `core/model/application/DefaultModelInvokeService.java` —— mock-chat 实现
5. `core/support/security/SecretEncryptor.java` —— 加密接口（阶段 1 用 Base64）
6. `web/controller/ModelController.java`

**核心知识点**：
```text
Mock First:    先写 mock-chat 跑通全链路，真实模型后续替换
API Key 加密:  SecretEncryptor 接口预留 → 阶段 1 Base64 → 阶段 2 AES/KMS
角色权限:     /api/admin/** → hasRole("ADMIN")，普通用户只能 GET /api/model-configs
```

### 第 5 天：看跨包调用怎么遵守纪律

**读这些文件**：
1. `core/profile/api/ProfileService.java` —— 接口
2. `core/profile/application/DefaultProfileService.java` —— **重点关注它调了什么**

```java
// DefaultProfileService 调了三个不同的包的 api：
applicationService.ensureApplicationOwned(...)    // core.identity.api
skillQueryService.areSkillsBindable(...)          // core.skill.api
mcpToolQueryService.areMcpToolsBindable(...)      // core.mcp.api

// 注意：它没有 import 任何 SkillMapper 或 McpToolMapper！
```

**核心知识点**：
```text
跨包调用规则:
  ✅ 可以调: core.xxx.api、core.xxx.dto、core.xxx.command
  ❌ 不能调: core.xxx.entity、core.xxx.mapper、core.xxx.internal
  ✅ Profile 想查 Skill 是否存在 → SkillQueryService（api 接口）
  ❌ Profile 想查 Skill 是否存在 → SkillMapper（mapper 不能碰）

ArchUnit 测试自动守卫这条规则，违反编译时测试失败。
```

### 第 6 天：看 Agent 怎么思考

**读这些文件**：
1. `core/agent/api/AgentRuntimeService.java` —— 接口
2. `core/agent/application/DefaultAgentRuntimeService.java` —— **ReAct 核心**
3. `core/context/api/AgentContextBuilder.java` —— 组装上下文
4. `core/context/application/DefaultAgentContextBuilder.java`
5. `core/skill/application/DefaultSkillExecutor.java` —— 内置 3 个 Skill
6. `core/mcp/application/DefaultMcpToolExecutor.java` —— mock MCP 工具

**核心知识点——ReAct 循环**：
```text
while (step < 3):
  1. 调模型 → 拿到文本回复
  2. 解析文本: @skill:calculator {json} → 调 SkillExecutor
              @mcp:read_file {json} → 调 McpToolExecutor
  3. 没工具调用 → 最终回复，退出循环
  4. 有工具调用 → 工具结果加回 messages → 循环

关键约束：
  工具执行前校验是否在 Context 授权列表内
  复用 conversationId 必须校验归属权
  AgentRuntimeService 只调 api 接口：
    SkillExecutor（skill.api）、ModelInvokeService（model.api）
    McpToolExecutor（mcp.api）、MemoryWriteService（memory.api）
```

### 第 7 天：看 SSE 怎么实时返回

**读这些文件（三跳链路）**：

```text
浏览器
  │ HTTP SSE
  ▼
1. web/controller/ChatController.java           ← 异步处理，返回 SseEmitter
2. web/client/HttpGatewayClient.java            ← HTTP 请求 Gateway，逐字节转发
  │ HTTP SSE (X-Internal-Token)
  ▼
3. gateway/controller/InternalAiController.java  ← 验内部 Token，转 GatewayChatRequest
4. gateway/controller/InternalAiController.streamTest() ← 固定 SSE 测试
  │ Maven 方法调用
  ▼
5. core/agent/api/AgentRuntimeService.run()      ← 同步执行，返回 AgentRunResult
```

**核心知识点**：
```text
SSE 不退缓冲的关键：
  StreamingResponseBody + output.flush() → 每行立即发
  HttpGatewayClient.copyStreaming() → 1024 字节缓冲区 + flush
  Gateway 出错 → ChatController 转成 SSE error 事件

三入口：
  GET  /api/chat/stream-test      → Web 转发 Gateway 固定 SSE（测试透传）
  POST /api/chat/stream            → Web 转发 Gateway（真实 Agent 对话）
  POST /api/ai/chat/stream         → Gateway API Key 入口（外部调用）

agent_mode:
  agent → 走 AgentRuntimeService（完整 ReAct）
  none  → 走 ModelInvokeService（纯模型转发）
  不指定 → 默认 agent
```
