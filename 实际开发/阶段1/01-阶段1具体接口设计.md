# 阶段 1 具体接口设计

> 本文档是开发落地版接口清单。正式接口边界以 `考核设计/接口设计/01-阶段1接口设计.md` 为准；本文只保留阶段 1 实现时需要逐个开发和联调的接口。

---

## 1. 开发原则

阶段 1 只做基础 Agent 闭环：

```text
注册 / 登录
Application / API Key
模型配置
Profile
Skill / MCP 查询和绑定
Web -> Gateway -> Core 对话
SSE 流式返回
会话消息保存
长期记忆写入
```

阶段 1 不做：

```text
完整 /v1/chat/completions
Trace 查询
Token 看板
安全策略配置
告警规则配置
Team 接口
完整前端控制台
```

## 2. Web API

### 2.1 Auth

| 方法 | 路径 | 说明 | 验收 |
|---|---|---|---|
| POST | `/api/auth/register` | 用户注册 | users 写入，默认 USER 角色 |
| POST | `/api/auth/login` | 用户登录 | 返回 JWT |
| GET | `/api/auth/me` | 当前用户 | JWT 可解析用户身份 |

### 2.2 Application / API Key

| 方法 | 路径 | 说明 | 验收 |
|---|---|---|---|
| POST | `/api/applications` | 创建应用并生成默认 API Key | applications、api_keys 写入；Key 明文只返回一次 |
| GET | `/api/applications` | 查询当前用户应用 | 分页返回 |
| GET | `/api/applications/{applicationId}/api-keys` | 查询 Key 列表 | 只返回 keyPrefix，不返回明文 |
| POST | `/api/applications/{applicationId}/api-keys/{apiKeyId}/revoke` | 吊销 Key | status 变为 REVOKED |

### 2.3 Model

| 方法 | 路径 | 说明 | 验收 |
|---|---|---|---|
| POST | `/api/admin/model-providers` | 创建模型供应商 | apiKey 加密存储 |
| POST | `/api/admin/model-configs` | 创建模型配置 | model_configs 写入 |
| GET | `/api/model-configs?status=ACTIVE` | 查询可用模型 | 登录用户可查询 |

阶段 1 通过 Flyway 初始化默认 Admin 用户，不开放公开 Admin 注册接口。

### 2.4 Profile

| 方法 | 路径 | 说明 | 验收 |
|---|---|---|---|
| POST | `/api/profiles` | 创建 Profile | agent_profiles 写入 |
| GET | `/api/profiles` | 查询 Profile 列表 | 支持 applicationId、分页 |
| GET | `/api/profiles/{profileId}` | 查询 Profile 详情 | 返回 Skill/MCP 绑定 |
| PUT | `/api/profiles/{profileId}/skills` | 绑定 Skill | profile_skills 写入 |
| PUT | `/api/profiles/{profileId}/mcp-tools` | 绑定 MCP Tool | profile_mcp_tools 写入 |

Profile 状态：

```text
DRAFT：创建者可测试和对话
PUBLISHED：可正式使用
DISABLED：不可用于对话
```

### 2.5 Skill / MCP

| 方法 | 路径 | 说明 | 验收 |
|---|---|---|---|
| GET | `/api/skills?scope=GLOBAL&status=ENABLED` | 查询可用 Skill | 至少返回 calculator/weather/search |
| GET | `/api/mcp-tools?status=AVAILABLE` | 查询可用 MCP Tool | 至少返回 1 个示例工具 |

### 2.6 Conversation / Message

| 方法 | 路径 | 说明 | 验收 |
|---|---|---|---|
| GET | `/api/conversations?applicationId=...` | 查询会话列表 | conversations 可查询 |
| GET | `/api/conversations/{conversationId}/messages` | 查询消息列表 | conversation_messages 可查询 |

### 2.7 Chat SSE

| 方法 | 路径 | 说明 | 验收 |
|---|---|---|---|
| GET | `/api/chat/stream-test` | Web 固定 SSE 透传测试 | curl -N 逐条输出事件 |
| POST | `/api/chat/stream` | 浏览器 AI 对话 | Web -> Gateway -> Core，SSE 返回 |

`/api/chat/stream` 请求字段：

```json
{
  "applicationId": 20001,
  "profileId": 50001,
  "conversationId": null,
  "message": "帮我计算 128 * 36 + 59",
  "enabledSkillIds": [60001],
  "enabledMcpToolIds": [70001],
  "stream": true
}
```

## 3. Web -> Gateway 内部接口

### 3.1 内部对话流

```http
POST /internal/ai/chat/stream
```

Header:

```text
X-Internal-Token: <internal-token>
X-Internal-Timestamp: <timestamp>
X-Internal-Nonce: <uuid>
Accept: text/event-stream
```

Request:

```json
{
  "tenantId": 1,
  "applicationId": 20001,
  "userId": 10001,
  "channel": "WEB",
  "agentMode": "agent",
  "profileId": 50001,
  "conversationId": null,
  "message": "帮我计算 128 * 36 + 59",
  "enabledSkillIds": [60001],
  "enabledMcpToolIds": [70001],
  "clientRequestId": "uuid-from-web"
}
```

验收：

```text
Gateway 校验 X-Internal-Token。
Gateway 生成 traceId。
Gateway 转成 AgentRunCommand。
Core 不读取 Gateway ThreadLocal。
```

### 3.2 Gateway 固定 SSE 测试流

```http
GET /internal/ai/stream-test
```

验收：

```powershell
curl.exe -N -H "X-Internal-Token: dev-internal-token" http://localhost:8081/internal/ai/stream-test
```

必须逐条看到：

```text
thinking
action
observation
message
done
```

## 4. Gateway API Key 最小入口

```http
POST /api/ai/chat/stream
```

Header:

```text
Authorization: Bearer sk-xxx
Accept: text/event-stream
```

`agentMode=agent`：

```json
{
  "agentMode": "agent",
  "profileId": 50001,
  "conversationId": null,
  "message": "用 calculator 计算 128 * 36 + 59",
  "enabledSkillIds": [60001],
  "enabledMcpToolIds": [70001],
  "stream": true
}
```

`agentMode=none`：

```json
{
  "agentMode": "none",
  "modelConfigId": 41001,
  "messages": [
    {
      "role": "user",
      "content": "你好"
    }
  ],
  "stream": true
}
```

规则：

```text
agentMode=agent 走 core.agent.AgentRuntimeService。
agentMode=none 走 core.model.ModelInvokeService。
agentMode=none 不走 AgentContextBuilder，不写 conversation_messages。
```

## 5. SSE 事件

事件类型：

```text
thinking
action
observation
message
done
error
```

通用结构：

```json
{
  "type": "message",
  "traceId": "tr_xxx",
  "conversationId": 80001,
  "step": 3,
  "content": "部分回复",
  "metadata": {}
}
```

要求：

```text
错误也尽量通过 error 事件返回。
不在事件里输出敏感原文。
Web 转发时不能缓冲。
浏览器断开后释放 Web/Gateway/Core 资源。
```

## 6. 最小联调顺序

```text
1. POST /api/auth/register
2. POST /api/auth/login
3. POST /api/applications
4. POST /api/admin/model-providers
5. POST /api/admin/model-configs
6. POST /api/profiles
7. GET /api/skills
8. GET /api/mcp-tools
9. GET /internal/ai/stream-test
10. GET /api/chat/stream-test
11. POST /api/chat/stream
12. GET /api/conversations
13. GET /api/conversations/{conversationId}/messages
```

工具：

```text
Apifox：普通 REST 接口。
curl.exe -N：SSE 流式接口。
DBeaver：检查数据库写入。
```
