# 阶段 1 测试指南

> 本文记录阶段 1 当前代码的推荐测试方式。目标是让你不用翻聊天记录，也能自己跑通 Web -> Gateway -> Core 的基础 Agent 闭环。

---

## 1. 测试分层

阶段 1 测试分三层：

```text
第一层：自动化测试
  验证代码编译、单元测试、Controller 测试、ArchUnit 架构边界。

第二层：固定 SSE 测试
  验证 Gateway SSE 和 Web SSE 透传没有被缓冲。

第三层：完整业务链路测试
  登录 -> 创建 Application -> 创建 Profile -> Web Chat SSE -> Gateway -> Core Agent。
```

建议顺序：

```text
先 mvn test
再跑 Gateway 固定 SSE
再跑 Web 固定 SSE
最后跑完整 Chat SSE
```

## 10. 完整 Chat SSE 测试

### 10.1 创建 Application

```powershell
$suffix = [DateTimeOffset]::Now.ToUnixTimeMilliseconds()

$appBody = @{
  name = "Step15 App $suffix"
  description = "Step 15 acceptance application"
} | ConvertTo-Json

$app = Invoke-RestMethod `
  -Uri "http://localhost:8080/api/applications" `
  -Method Post `
  -Headers @{"Authorization"="Bearer $token"} `
  -ContentType "application/json" `
  -Body $appBody

$applicationId = $app.data.applicationId
$applicationId
```

不要硬写 `applicationId=1`。正确做法是先创建 Application，再用返回的 ID。

### 10.2 创建 Profile

```powershell
$profileBody = @{
  applicationId = $applicationId
  name = "Step15 Default Agent $suffix"
  profileType = "ASSISTANT"
  description = "Step 15 acceptance profile"
  modelConfigId = 1
  promptExtra = "You are a concise assistant."
  memoryStrategy = $null
  maxSteps = 3
  visibility = "PRIVATE"
} | ConvertTo-Json -Depth 5

$profile = Invoke-RestMethod `
  -Uri "http://localhost:8080/api/profiles" `
  -Method Post `
  -Headers @{"Authorization"="Bearer $token"} `
  -ContentType "application/json" `
  -Body $profileBody

$profileId = $profile.data.profileId
$profileId
```

注意字段名是：

```text
profileId
```

不是：

```text
id
```

### 10.3 发起 Chat SSE

```powershell
$chatBody = @{
  applicationId = $applicationId
  agentMode = "agent"
  profileId = $profileId
  conversationId = $null
  message = "hello"
  enabledSkillIds = @()
  enabledMcpToolIds = @()
  clientRequestId = "step15-$suffix"
  modelConfigId = 1
  messages = $null
  stream = $true
} | ConvertTo-Json -Depth 5

curl.exe -N `
  -X POST `
  -H "Authorization: Bearer $token" `
  -H "Accept: text/event-stream" `
  -H "Content-Type: application/json" `
  -d $chatBody `
  http://localhost:8080/api/chat/stream
```

当前成功结果应包含：

```text
event: thinking
event: message
event: done
```

示例最终消息：

```text
[mock-chat] Echo: hello
```

## 11. 用 Apifox 怎么测

Apifox 适合测普通 REST 接口：

```text
POST /api/auth/login
POST /api/applications
POST /api/profiles
GET /api/skills
GET /api/mcp-tools
```

Apifox 也能发 SSE 请求，但阶段 1 推荐优先用 `curl.exe -N` 测 SSE，因为它能直接看出是不是逐事件返回。

```text
Authorization: Bearer <JWT>
Accept: text/event-stream
Content-Type: application/json
```

## 12. 用 DBeaver 看哪些表

完整 Chat SSE 后，可以看这些表：

```sql
select * from applications order by id desc;
select * from api_keys order by id desc;
select * from agent_profiles order by id desc;
select * from conversations order by id desc;
select * from conversation_messages order by id desc;

```

重点确认：

```text

agent_profiles 有刚创建的 Profile。
conversations 有新会话。
conversation_messages 有 user 和 assistant 消息。
memories 有 SUMMARY 类型记录。
api_keys 只存 hash 和 prefix，不存完整明文。
```

## 13. 已知限制

当前完整 Chat SSE：

```text
thinking -> message -> done
```

暂时看不到 ReAct 中间步骤：

```text
thinking -> action -> observation -> message -> done
```

原因：

```text
AgentRuntimeService.run() 当前是同步方法。
Gateway 只能在调用前写 thinking，在调用完成后写 message/done。
Agent 内部工具调用过程还没有 listener / callback / event sink。
```

这不是 Web SSE 透传失败。Web 和 Gateway 的 SSE 链路已经通过固定流测试验证。

后续增强：

```text
给 AgentRuntimeService 增加事件回调。
Runtime 在 context building / model call / tool call / tool result / final message 时推事件。
Gateway 将 Runtime 事件转换成 SSE。
Web 继续边读边写透传。
```

## 14. 接入真实模型测试

阶段 1 默认 seed 数据使用 `mock-chat`，不用 API Key 也能跑通主链路。现在 `core.model` 已支持 `OPENAI_COMPATIBLE` 供应商，真实模型可以通过后台接口配置。

推荐先接 DeepSeek：

```text
provider_type: OPENAI_COMPATIBLE
base_url: https://api.deepseek.com/v1
model_name: deepseek-chat
```

### 14.1 创建真实模型供应商

用 admin JWT 调用：

```powershell
$providerBody = @{
  name = "DeepSeek"
  providerType = "OPENAI_COMPATIBLE"
  baseUrl = "https://api.deepseek.com/v1"
  apiKey = "你的 DeepSeek API Key"
} | ConvertTo-Json

$provider = Invoke-RestMethod `
  -Uri "http://localhost:8080/api/admin/model-providers" `
  -Method Post `
  -Headers @{"Authorization"="Bearer $token"} `
  -ContentType "application/json" `
  -Body $providerBody

$providerId = $provider.data.providerId
$providerId
```

API Key 会通过 `SecretEncryptor` 加密后写入数据库，创建响应不会返回明文。

### 14.2 创建真实模型配置

```powershell
$modelBody = @{
  providerId = $providerId
  modelName = "deepseek-chat"
  displayName = "DeepSeek Chat"
  capabilitiesJson = '{"text":true,"stream":false}'
  defaultTemperature = 0.7
  maxContextTokens = 8192
} | ConvertTo-Json

$model = Invoke-RestMethod `
  -Uri "http://localhost:8080/api/admin/model-configs" `
  -Method Post `
  -Headers @{"Authorization"="Bearer $token"} `
  -ContentType "application/json" `
  -Body $modelBody

$realModelConfigId = $model.data.modelConfigId
$realModelConfigId
```

### 14.3 用真实模型创建 Profile

创建 Profile 时把 `modelConfigId` 改成真实模型 ID：

```powershell
$profileBody = @{
  applicationId = $applicationId
  name = "DeepSeek Agent $suffix"
  profileType = "ASSISTANT"
  description = "Real model acceptance profile"
  modelConfigId = $realModelConfigId
  promptExtra = "You are a concise assistant."
  memoryStrategy = $null
  maxSteps = 3
  visibility = "PRIVATE"
} | ConvertTo-Json -Depth 5

$profile = Invoke-RestMethod `
  -Uri "http://localhost:8080/api/profiles" `
  -Method Post `
  -Headers @{"Authorization"="Bearer $token"} `
  -ContentType "application/json" `
  -Body $profileBody

$realProfileId = $profile.data.profileId
$realProfileId
```

### 14.4 用真实模型发起 Chat SSE

```powershell
$chatBody = @{
  applicationId = $applicationId
  agentMode = "agent"
  profileId = $realProfileId
  conversationId = $null
  message = "用一句话介绍你自己"
  enabledSkillIds = @()
  enabledMcpToolIds = @()
  clientRequestId = "real-model-$suffix"
  modelConfigId = $realModelConfigId
  messages = $null
  stream = $true
} | ConvertTo-Json -Depth 5

curl.exe -N `
  -X POST `
  -H "Authorization: Bearer $token" `
  -H "Accept: text/event-stream" `
  -H "Content-Type: application/json" `
  -d $chatBody `
  http://localhost:8080/api/chat/stream
```

成功时仍然会看到：

```text
event: thinking
event: message
event: done
```

区别是 `message` 不再是：

```text
[mock-chat] Echo: hello
```

而是真实模型返回的内容。

### 16.3 PowerShell 中文乱码说明

PowerShell 中看到类似：

```text
ä½ å¥½...
```

这是 PowerShell 控制台编码显示问题，不代表后端或模型返回内容错误。浏览器前端正常按 UTF-8 解析 SSE 时，中文一般不会乱码。后续前端实现时仍建议显式使用 UTF-8 解码。

### 16.4 当前流式能力边界

当前 SSE 是事件级流式：

```text
thinking -> message -> done
```

还不是 token 级流式：

```text
delta -> delta -> delta -> done
```

真实模型当前请求参数仍是：

```json
{"stream": false}
```

因此模型内容会一次性出现在 `message` 事件中。Token 级流式输出放到阶段 2，通过 `ModelInvokeService` streaming + `AgentRuntime` event sink 实现。
