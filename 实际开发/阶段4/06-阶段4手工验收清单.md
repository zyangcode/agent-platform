# 阶段 4 手工验收清单

## 1. 验收目标

阶段 4 验收目标是确认 Agent Team 最小闭环可用：

- Profile 可以选择 `TEAM` 执行模式。
- Chat 仍使用原 `/api/chat/stream` 入口，按 Profile 的 `executionMode` 自动进入 Team Runtime。
- SSE 能实时看到 `team_*` 阶段事件，而不是全部执行完后一次性出现。
- 右侧 Team 面板能展示计划、任务状态、Reviewer 和 Mermaid 文本。
- 如果 Reviewer 要求补充新信息，Team 可进入一次 re-plan，并只执行新增任务。
- Trace Detail 能看到 Team spans，并展示 Team 相关属性。
- Token Usage 不因 Team 模式断链。
- BASIC 单 Agent 模式不回归。

阶段 4 不验收 token 级逐字流式输出、流程图编辑器、完整 Mermaid 图形渲染、Team 数据表、并发 Team 执行、高危工具确认、Jar Skill 上传 UI。

## 2. 启动前检查

1. 启动 PostgreSQL。
2. 启动 Web：`http://localhost:8080`。
3. 启动 Gateway：`http://localhost:8081`。
4. 启动前端：在 `agent-platform-frontend` 执行 `npm run dev -- --host 127.0.0.1`，访问 `http://127.0.0.1:5173`。

自动化验证先执行：

```powershell
mvn.cmd -s .mvn/settings.xml test
npm --prefix agent-platform-frontend run build
```

验收标准：Maven 全量测试通过；前端 build 通过。Vite chunk size warning 当前已知，可接受。

## 3. 创建 Team Profile

1. 登录前端。
2. 进入 Applications，确认至少有一个 Application。
3. 进入 Profiles，创建或编辑一个 Profile。
4. `Execution mode` 选择 `Team Agent`。
5. 选择可用模型配置，填写风格提示词和 max steps。
6. 保存后重新打开该 Profile。

验收标准：

- Profile 详情中能看到 `Team Agent`。
- 刷新页面后执行模式仍为 `Team Agent`。
- 保存 Profile 不出现 Unauthorized 或 4xx。

## 4. 绑定工具

1. 在 Profile 详情下方 Tool bindings 勾选可用 Skill/MCP Tool。
2. 点击 `Save bindings`。
3. 重新打开 Profile。

验收标准：

- 已绑定 Skill/MCP Tool 显示在上方 bindings 区域。
- 下方编辑区勾选状态和保存结果一致。
- 没有工具时，Team 仍可通过通用 `MODEL_TASK` fallback 回答普通问题。

## 5. Team Chat 主链路

1. 进入 Agent Chat。
2. 选择刚才的 Application。
3. 选择 `Agent` 模式。
4. 选择 `executionMode=TEAM` 的 Profile。
5. 发送：

```text
帮我策划一个本周六下午在重庆适合 20 人的团建活动，预算 3000 元以内，最好不要太累。
```

验收标准：

- 聊天最终有 assistant 回复。
- 右侧 Runtime 中出现 Team run 面板。
- Event timeline 中能看到 `team_start`、`team_plan`、`team_task_start`、`team_task_result`、`team_review`、`team_final`、`message`、`done`。
- Team run 面板中能看到 Goal、任务状态、Reviewer、Mermaid 文本。
- Team 事件应在执行过程中逐步出现，不应等最终回答出来后一次性出现。

## 6. 普通问题 fallback

继续使用 Team Profile，发送：

```text
1+1 等于几？
```

验收标准：

- 不要求 Planner 一定拆出多任务。
- 如果模型规划失败，系统应降级为通用单任务 `MODEL_TASK`。
- 最终能给出正常回答，不应因为缺少 weather/search/calculator 而失败。

## 7. BASIC 模式回归

1. 创建或选择一个 `Basic Agent` Profile。
2. 在 Chat 中选择该 Profile。
3. 发送一个简单问题。

验收标准：

- 可以正常回复。
- Runtime 右侧不显示 Team run 面板。
- Event timeline 仍是原基础事件，如 `thinking/action/observation/message/done`。

## 8. Trace Detail

1. 在 Team Chat 完成后复制或点击本次 Trace ID。
2. 进入 Traces，打开对应 Trace Detail。
3. 查看 Span timeline 和 Span detail。

验收标准：

- 能看到 `team.run`、`team.plan`、`team.task.execute`、`team.review` 等 Team span。
- 页面显示为中英文友好的名称，例如 `Team 运行`、`Team 任务执行`。
- Span detail 能展示已有的 `taskId`、`taskType`、`profileId` 等结构化属性。
- Attributes 长 JSON 不应重叠或撑破面板。

## 9. Token Usage

1. 打开 Token Usage 页面。
2. 选择 Team Chat 使用的 Application。
3. 查看列表和汇总。

验收标准：

- Team 运行产生的 Planner / Executor MODEL_TASK / Reviewer 模型调用能进入 token usage 记录。
- Reviewer re-plan 场景中，第二次 Planner 模型调用也能进入 token usage 记录。
- 有真实 usage 时显示真实 token；没有真实 usage 时可显示 estimated。
- 页面不因 Team spanId 或多次模型调用崩溃。

## 10. 错误与降级

可选验证：

- 临时使用 mock 模型或让模型返回非法 JSON，确认 Planner 会重试一次并 fallback。
- 让 Reviewer 返回非法 JSON，确认 fallback warning 不阻断最终输出。
- 让 Reviewer 返回 `replanRequired=true` 且带 `replanInstruction`，确认事件流出现 `team_retry` 和第二次 `team_plan`，并且旧 task 不重复执行。
- 让工具失败，确认对应 task 标记 failed，Team 不直接崩溃。

验收标准：

- 失败路径有 `error` 或 failed 状态，不白屏。
- Team Runtime 不因为单个工具失败直接中断。
- Gateway/Web SSE 仍能结束流。
