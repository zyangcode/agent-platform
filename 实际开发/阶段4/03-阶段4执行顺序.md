# 阶段 4 执行顺序

> 本文用于阶段 4 实际开发推进。设计边界见：`实际开发/阶段4/01-阶段4AgentTeam设计说明.md`。接口细节见：`实际开发/阶段4/02-阶段4接口设计.md`。

## 1. 总体目标

阶段 4 实现 Agent Team 最小稳定闭环：

```text
Profile executionMode=TEAM
  -> Chat 请求
  -> Planner 生成 TaskPlan
  -> Executor 调用授权 Skill/MCP
  -> Reviewer 审查
  -> Orchestrator 汇总最终答案
  -> SSE 展示过程
  -> Trace 展示 Team span
```

阶段 4 不做：

- 新 Team 数据表。
- 工作流设计器。
- 并发执行。
- 复杂 Pipeline 重构。
- 高危工具确认。
- token 级真实流式输出。

## 2. 开发原则

1. 先后端 Team Runtime，后前端展示。
2. 先 mock/规则兜底跑通活动策划 Demo，再接真实模型 JSON 输出。
3. 先复用现有 Skill/MCP 执行接口，再抽 AgentTool 统一视图。
4. Team 数据先落 Trace Span attributes，不新增表。
5. 每个 Step 必须有测试和可验证结果。

## 6. Step 4：实现 Planner

目标：

- Planner 根据用户输入、Profile Prompt、AgentTool 列表生成 TaskPlan。
- 第一版必须有稳定兜底，保证活动策划 Demo 不依赖真实模型 JSON 稳定性。

新增文件：

```text
agent-platform-core/src/main/java/com/ls/agent/core/team/api/TeamPlanner.java
agent-platform-core/src/main/java/com/ls/agent/core/team/command/PlanTeamCommand.java
agent-platform-core/src/main/java/com/ls/agent/core/team/application/DefaultTeamPlanner.java
```

策略：

```text
1. 优先调用模型生成 TaskPlan JSON。
2. JSON 解析或校验失败时，要求模型修正一次。
3. 仍失败时，如果可用工具包含 weather/search/calculator，则生成活动策划预置 TaskPlan。
4. 如果工具不足，生成单任务 TaskPlan，由模型直接总结。
```

测试：

```text
DefaultTeamPlannerTest
```

重点测试：

- 模型返回合法 JSON。
- 模型第一次返回非法 JSON，第二次修正成功。
- 两次都失败时使用 fallback plan。

提交建议：

```text

```

## 7. Step 5：实现 Executor

目标：

- 按依赖顺序执行任务。
- 对 suggestedTool 不为空的任务调用 AgentToolDispatcher。
- 没有 suggestedTool 的任务走模型总结或标记为 success。

新增文件：


规则：

```text

工具调用成功 -> success。
工具调用失败 -> failed，但不中断整体。
每个任务写 trace span。
每次工具调用写 team.tool.call span。
```

## 10. Step 8：Gateway/Web SSE 输出 Team 事件

目标：

- Core Team Runtime 产生 Team 事件。
- Gateway/Web SSE 能把事件转给前端。

当前 `AgentRunResult` 只在完成后返回工具事件，阶段 4 有两种方案：

方案 A：低风险

```text
Team Runtime 内部收集 TeamEventDTO。
AgentRunResult 增加 events。
Gateway 在 run 完成后按顺序输出 team_* 事件，再输出 message/done。
```

方案 B：更实时

```text
AgentRunCommand 增加 AgentEventSink。
Team Runtime 执行过程中实时 emit。
Gateway 边执行边写 SSE。
```

阶段 4 第一版建议方案 A，原因是改动小，和当前事件级 SSE 一致。真正 token 级/实时 Team 事件后续单独做。

涉及文件：

```text
agent-platform-core/src/main/java/com/ls/agent/core/agent/dto/AgentRunResult.java
```
