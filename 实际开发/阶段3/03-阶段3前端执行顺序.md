# 阶段 3 前端执行顺序

> 本文用于阶段 3 实际开发推进。更完整的页面设计说明见：`实际开发/阶段3/02-阶段3前端控制台设计说明.md`。

## 1. 阶段目标

阶段 3 要完成一个可演示的前端控制台 MVP，让后端阶段 1、阶段 2 已有能力能被用户在浏览器里操作和观察。

MVP 完成标准：

- 能登录并保持 JWT 会话。
- 能创建、查看 Application，并看到创建时返回的一次性 API Key。
- 能创建、查看 Profile，并绑定已有 Skill / MCP 工具。
- 能通过 Web 入口发起 Agent Chat，消费 SSE 事件。
- 能查看 Trace 列表和 Trace 明细。
- 能查看 Token Usage 汇总和明细。
- 能展示已有 Skill / MCP 工具。
- 页面有明确的 loading、empty、error、401、403、429 状态。
- 本地 `npm run build` 通过，主要页面完成浏览器手工验收。

阶段 3 暂不做：

- Skill 上传和 Jar 热加载 UI。
- Team 编排页面。
- 完整 Admin 用户管理。
- 完整模型供应商管理 UI。
- 完整安全策略和告警规则配置 UI。
- nginx 部署配置。

## 4. 后端接口基线

所有普通 JSON API 都返回：

```ts
type ApiResponse<T> = {
  success: boolean
  code: string
  message: string
  data: T
}
```

分页接口返回：

```ts
type PageResult<T> = {
  records: T[]
  pageNo: number
  pageSize: number
  total: number
  totalPages: number
}
```

当前可用接口：

| 功能 | 方法 | 路径 |
|---|---|---|
| 注册 | POST | `/api/auth/register` |
| 登录 | POST | `/api/auth/login` |
| 当前用户 | GET | `/api/auth/me` |
| 创建 Application | POST | `/api/applications` |
| Application 列表 | GET | `/api/applications?pageNo=1&pageSize=20` |
| API Key 列表 | GET | `/api/applications/{applicationId}/api-keys` |
| 撤销 API Key | POST | `/api/applications/{applicationId}/api-keys/{apiKeyId}/revoke` |
| 模型配置列表 | GET | `/api/model-configs` |
| 创建 Profile | POST | `/api/profiles` |
| Profile 列表 | GET | `/api/profiles?applicationId={id}&pageNo=1&pageSize=20` |
| Profile 详情 | GET | `/api/profiles/{profileId}` |
| 绑定 Skill | PUT | `/api/profiles/{profileId}/skills` |
| 绑定 MCP Tool | PUT | `/api/profiles/{profileId}/mcp-tools` |
| Skill 列表 | GET | `/api/skills?scope=&status=` |
| MCP Tool 列表 | GET | `/api/mcp-tools?status=` |
| Chat SSE | POST | `/api/chat/stream` |
| Trace 列表 | GET | `/api/traces` |
| Trace 详情 | GET | `/api/traces/{traceId}` |
| Token Usage 列表 | GET | `/api/token-usages` |
| Token Usage 汇总 | GET | `/api/token-usages/summary` |

Chat 请求体字段：

```ts
type ChatStreamRequest = {
  applicationId?: number
  agentMode?: 'agent' | 'none'
  profileId?: number
  conversationId?: number
  message?: string
  enabledSkillIds?: number[]
  enabledMcpToolIds?: number[]
  clientRequestId?: string
  modelConfigId?: number
  messages?: Array<{ role: string; content: string }>
  stream?: boolean
}
```

阶段 3 默认使用：

```json
{
  "agentMode": "agent",
  "stream": true
}
```

已知阶段 2 遗留问题：

- `MODEL_ERROR` 场景下 `trace_roots` 可能没有写入。
- 前端 Trace 相关页面必须把 Trace 缺失作为正常空状态处理，不能崩溃。

## 5. 推荐前端目录

```text
agent-platform-frontend/
  src/
    app/
      router.tsx
      providers.tsx
    components/
      layout/
      ui/
      charts/
      feedback/
    features/
      auth/
      dashboard/
      applications/
      profiles/
      chat/
      traces/
      token-usage/
      tools/
    lib/
      api/
      auth/
      sse/
      format/
    styles/
```

边界：

- `lib/api`：HTTP Client、响应解包、错误标准化、JWT Header 注入。
- `lib/auth`：Token 存取、当前用户状态、登录态恢复。
- `lib/sse`：SSE 解析和错误收敛。
- `features/*`：页面级组件和对应接口封装。
- `components/ui`：shadcn/ui 或项目统一基础组件。
- `components/layout`：控制台 Shell、侧边栏、顶部状态栏。
- `components/feedback`：Loading、Empty、Error、Unauthorized、QuotaExceeded。

### Step 10：实现 Agent Chat 和 SSE

目标：

- 用户选择 Application、Profile、Skill、MCP 后发起对话。

- 右侧运行详情展示 Trace ID、状态、Token、工具事件。

主要接口：

```text
POST /api/chat/stream
```

请求默认：

```json
{
  "agentMode": "agent",
  "stream": true
}
```

SSE 事件类型：

```text
thinking
action
observation
message
done
error
```

主要文件：


重点：

- `429` 显示配额不足，不当作普通 500。
- 敏感数据被拦截时显示安全策略提示。

验收：

- 能发起一次 Agent 对话。
- 能看到流式 message。
- 能看到 thinking/action/observation 等事件。
