# 阶段 3 验收总结

## 1. 当前结论

阶段 3 前端控制台 MVP 已完成代码级收口，当前状态为“可进入浏览器手工验收”。

本阶段目标不是完整 Admin 后台，也不是完整 Skill 市场，而是让阶段 1、阶段 2 已有能力可以在浏览器中完成 Demo：

- 登录 / 注册
- Application / API Key 管理
- Profile 创建、编辑、禁用和工具绑定
- Agent Chat 和 Direct model Chat
- 历史会话新建、加载和单条归档删除
- Tools catalog 查询、筛选和搜索
- Trace 列表、详情、Span 明细和分页
- Token Usage 汇总、明细、Top models 和分页
- Admin Model Configs
- Admin Security / Users 只读收口页
- 主要页面中文 / EN 切换

## 2. 已完成能力

### 2.1 前端工程基础

- React + Vite + TypeScript + Tailwind CSS + shadcn/ui 风格组件。
- Vite dev server 代理 `/api` 到 Web 后端。
- 路由级 `ErrorBoundary`。
- JWT 登录态保存和 401 清理。
- 统一 `apiClient` 和 SSE 解析工具。
- 统一 loading / empty / error 状态。

### 2.2 权限与路由

- 未登录访问受保护页面会跳转 `/login`。
- 左侧导航按角色隐藏 Admin 菜单。
- Admin 路由已增加前端 `RoleGuard`。
- 后端 `/api/admin/**` 仍由 Spring Security `hasRole("ADMIN")` 保护。

### 2.3 Application / API Key

- 创建 Application。
- 创建后一次性展示 API Key 明文。
- 刷新后只展示 `keyPrefix`。
- API Key 可撤销。
- Application 支持编辑名称/描述。
- Application 删除语义为禁用，禁用后仍可审计查看。

### 2.4 Profile / Tool Binding

- 按 Application 创建 Profile。
- Profile 支持编辑模型、风格提示词和 max steps。
- Profile 支持禁用。
- 详情页展示模型、风格提示词、Skill 绑定、MCP 绑定。
- Tool bindings 支持勾选 Skill / MCP Tool 并保存。
- `promptExtra` 前端展示名统一为“风格提示词 / Style prompt”。

### 2.5 Chat

- 支持 Agent 模式。
- 支持 Direct model 模式。
- 支持选择 Application、Profile、Model Config。
- 使用 `fetch + ReadableStream + TextDecoder` 消费 Web SSE。
- 流式期间禁止重复提交。
- 支持 Stop 中断。
- Runtime 面板展示事件、状态、Trace ID、Conversation。
- 支持当前页面会话 sessionStorage 恢复。
- 支持历史会话列表、加载历史消息、新对话、单条归档删除。

### 2.6 Observability

- Trace 页面支持 Application / Status / Entrypoint 筛选。
- Trace 列表支持分页。
- Trace 详情展示 Root、Span timeline、Span detail、Token usage records。
- 长 Trace ID、错误信息和 JSON attributes 已做换行/滚动处理，避免重叠。
- Trace ID / Span ID 支持复制。
- Token Usage 页面支持 Application / Model Config 筛选。
- Token Usage 支持汇总、Prompt / Completion 拆分、Top models、明细和分页。

### 2.7 Tools

- 展示已有 Skills。
- 展示已有 MCP Tools。
- 支持 Skill scope / status 筛选。
- 支持 MCP status 筛选。
- 支持本地搜索名称、编码、描述、状态。
- 明确不提供阶段 3 Jar Skill 上传入口。

### 2.8 Admin

- Admin Model Configs 可创建 OpenAI-compatible Provider。
- Admin Model Configs 可创建 Model Config。
- Admin Security 是只读收口页，展示 Gateway 治理链、敏感扫描规则、安全事件审计和暂缓项。
- Admin Users 是只读收口页，展示当前账号、角色模型和暂缓项。

### 2.9 中英文切换

- 顶部状态栏支持 `中文 / EN` 切换。
- 语言选择写入 `localStorage`，刷新后保持。
- 已覆盖导航、顶部栏、Dashboard、Applications、Profiles、Chat、Tools、Traces、Token Usage、Admin 主要文案。
- 后端枚举和技术名保持原样，例如 `ACTIVE`、`ENABLED`、`OPENAI_COMPATIBLE`、`API Key`、`Trace ID`、`Schema`。

## 4. 已知非阻塞项

### 4.1 Chat 是事件级 SSE，不是 token 级流式

当前浏览器可以收到 SSE 事件，但后端模型调用仍以完整 message 为主，不是逐 token delta 输出。

后续增强方向：

- `core.model` provider 支持真实 `stream=true`。
- Agent Runtime 增加 event sink / callback。
- Gateway 将 token delta、tool event、final message 统一转为 SSE。

### 4.2 ECharts 已安装但暂未使用

Token Usage 当前使用 CSS 条形图和基础表格。阶段 3 MVP 已满足基础用量看板，不强行引入 ECharts，避免增加体积和复杂度。

后续如果做趋势图、成本折算、多维统计，再接入 ECharts。

### 4.3 Vite chunk size warning

当前 bundle 超过 500KB 提醒，运行不受影响。

后续可做：

- 路由级 lazy loading。
- 按需拆分 Admin / Chat / Observability 页面。
- 清理暂未使用的大依赖。

### 4.4 Admin 仍不是完整后台

当前 Admin 页面满足 MVP 展示和基础模型配置：

- Model Configs 可用。
- Security / Users 是只读收口页。

暂缓内容：

- 用户 CRUD。
- 角色分配。
- 用户禁用 / 重置密码。
- 安全策略 CRUD。
- 告警策略配置。
- 安全事件检索后台。

### 4.5 Skill 上传 / Jar 热加载 UI 未实现

阶段 3 Tools 只展示和筛选已有 Skill / MCP Tool。

Jar Skill 热加载是后续 Skill 市场增强，需要单独设计：

- 上传
- 校验
- 加载
- 禁用 / 卸载
- 审核
- 回滚
- 审计

### 4.6 MODEL_ERROR 时 Trace root 可能缺失

这是阶段 2 已知后端路径问题。阶段 3 前端已将 Trace 缺失作为正常 empty / error 状态处理，不会崩溃。
