# 阶段 3 前端开发计划

## 1. 阶段目标

阶段 3 目标是完成前端控制台 MVP，让项目从“接口可用”变成“可以演示”：

- 能登录。
- 能创建和查看 Application / API Key。
- 能创建和查看 Agent Profile。
- 能发起 Agent 对话并展示 SSE 返回。
- 能查看 Trace 列表和 Trace 详情。
- 能查看 Token Usage 明细和汇总。
- 能展示 Skill / MCP 工具基础信息。

本阶段不追求完整后台管理系统，不做全量 RBAC 管理、不做复杂租户管理、不做完整告警规则配置。

## 2. 技术栈

固定使用：

```text
React
Vite
TypeScript
Tailwind CSS
shadcn/ui
ECharts
```

前端工程目录：

```text
agent-platform-frontend/
```

本地开发：

```text
Frontend: http://localhost:5173
Web:      http://localhost:8080
Gateway:  http://localhost:8081
```

Vite 代理：

```text
/api -> http://localhost:8080
```

nginx 暂不进入阶段 3 开发；最终 Docker Compose / 部署交付阶段再补。

## 3. 审美选择规则

涉及页面颜值和交互体验时，先问用户，再实现。

必须先确认的选择：

- 整体视觉风格。
- 颜色主题。
- 登录页方向。
- 主导航形态。
- Dashboard 信息密度。
- Agent 对话页布局。
- Trace 详情页展示方式。
- Token 图表风格。
- 动效强度。
- 卡片、表格、弹窗等组件视觉密度。

每次确认时给 2-3 个方案，说明取舍，然后等用户选择。

可以直接执行的内容：

- 创建前端工程。
- 安装固定技术栈依赖。
- 配置 Vite 代理。
- 配置 Tailwind / shadcn/ui。
- 建立 API Client。
- 建立路由骨架。
- 定义 TypeScript 类型。
- 封装认证 Token 存取。

## 4. 页面范围

第一轮 MVP 页面：

```text
1. Login
2. Console Dashboard
3. Applications / API Keys
4. Profiles
5. Agent Chat
6. Trace List
7. Trace Detail
8. Token Usage
9. Skills / MCP Tools
```

暂缓页面：

```text
Admin User Management
完整模型供应商管理 UI
完整安全策略配置 UI
完整告警规则配置 UI
完整 Skill 上传审核 UI
Team 编排页面
飞书机器人配置页面
```

## 5. 推荐执行顺序

```text
Step 1: 确认整体视觉方向和页面信息架构
Step 2: 创建 agent-platform-frontend 工程
Step 3: 配置 Tailwind、shadcn/ui、路由、API Client
Step 4: 实现登录和认证态保持
Step 5: 实现 Console Layout 和导航
Step 6: 实现 Application / API Key 页面
Step 7: 实现 Profile 页面
Step 8: 实现 Agent Chat 页面和 SSE 消费
Step 9: 实现 Trace List / Trace Detail
Step 10: 实现 Token Usage 页面和图表
Step 11: 实现 Skills / MCP Tools 展示
Step 12: 联调、截图检查、阶段 3 手工验收
```

## 6. 第一轮需要用户确认的问题

开始写页面前先确认：

```text
1. 整体风格：暗色 AI 控制台 / 明亮 SaaS 控制台 / 极简工程后台
2. 导航形态：左侧固定导航 / 顶部导航 + 侧栏 / 类 IDE 三栏布局
3. Dashboard 密度：数据大屏感 / 工作台感 / 管理后台表格优先
```

确认后再进入前端工程创建和页面实现。
